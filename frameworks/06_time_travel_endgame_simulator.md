# 06 时空穿越者式极限思索（T0/Now）× 盡頭推演引擎（Endgame Simulator）

> 目标：用**两次视角切换（T0→Now）** + **闸门（Gate 1/2/3）** + **Endgame Map（先 Catastrophic）**，在不靠事后诸葛的前提下，识别“BTC 级（10x~1000x 非线性）”机会，并把输出变成**决策工程**：假设 → 探针 → 分段下注 → 退出/重启。
>
> 适用：你给一篇文章/线索，直觉像“新 Bitcoin”但主流尚未理解；你要的是**可执行策略**与**可证伪更新**。  
> 不适用：纯短线交易；或你无法/不愿做任何取证与复盘（那只能讲故事）。

---

## 核心原则（先锁灾难上界，再谈收益）

- **先 Catastrophic bound**：先写“不可逆红线”，再写上行空间。  
- **每个关键断言三件套**：证据（可追溯）/ 反证条件（可观测）/ 下一步取证（探针）。  
- **禁止事后诸葛**：T0 只用当时合理可得信息；Now 才允许利用“今天走到哪”。  
- **时变信息强制核验**：价格/市值/链上/用户/融资/政策/版本/人物职位等 → 必须 web search 并标注日期；否则不得 Gate 3。  

### 三视角切换（借鉴：穿越者 / 机制 / 上帝）

这不是文风，而是“强制换镜头”的防偏差机制：

- **穿越者视角（T0）**：只用当时可得信息与约束（对应第 1 节）。  
- **机制视角**：用经济学/博弈论/社会动力学拆解激励、参与者与传播（主要落在第 2/2.5/3/5/8 节）。  
- **上帝视角**：看穿盲区，给出最短通关路径与最短反证路径（主要落在第 6/8 节）。  

---

## 三闸门（只能三选一）

- **Gate 1 淘汰**：反证路径太短 / 红线风险过高 / 无可验信号（Time-to-Signal 太长且不可控）。  
- **Gate 2 买信息期权**：小成本换关键证据（探针+时间盒+退出阈值），先把 20% 推到 50%。  
- **Gate 3 分段下注**：已具备清晰升维信号（20→50→80 的升级条件写死），且 Catastrophic bound 可控。  

---

## 固定输出格式（每次必交付，不可省略标题）

> 下面是“骨架模板”。你每次输出都必须保留这些标题；内容允许短，但不能把标题省掉。

### 0) 快速结论（1段）
- Bitcoin 级潜力：A / B / C / D
- 当下决策：Gate 1 / Gate 2 / Gate 3
- 一句理由（只写最核心那条）

### 1) T0 时空重建（当时环境）
- 材料定性（先贴标签，避免拿错工具）：类型/阶段/可信度/时效（可复用 `frameworks/05_reasoning_toolkit.md` 的 A1–A3）
- T0：技术 / 基建 / 监管 / 资金 / 社会心理（每项 1–3 点）
- 当时大多数人忽略/误解什么？
- 当时最大的摩擦/阻力在哪？

### 2) 第一性原理：它解決什麼「不可妥協問題」？
- Problem：谁痛？痛到什么程度？为什么现有方案不可行？
- Primitive（新原语）：它创造了什么新能力？（一句话）
- Necessities（必要条件×3）：缺任何一个就不可能成功
- Flywheel（技术/经济飞轮）：用户→价值→安全/可用性→用户（用箭头链条写）

### 2.5) 社会底座与合法性（Belief Flywheel，必输）
- S1 合法性裂缝（Legitimacy Vacuum）：承接哪条制度裂缝？扩大还是缩小？
- S2 最早部落（Tribe）：谁会先信？身份动机？愿意付出什么代价？
- S3 神话与仪式（Myth/Ritual）：一句话神话 + 可重复节律/符号
- S4 敌对叙事（Narrative Enemy）：最强反叙事是什么？如何把攻击反脆弱化？
- S5 协调红利（Coordination Gain）：降低哪种协调成本？让陌生人一起做到什么？
- S6 信念钩子（Belief Hook，仅 1 条）：为什么能长期持有/建设/忍受波动？
  - 若缺 S6：**不得 Gate 3**，只能 Gate 2
- Belief Flywheel：裂缝/不信任→象征/身份→传播→社会证明/资本→更强信念（箭头链条）
- 耦合点：技术/经济飞轮与信念飞轮在哪里耦合？（例：价格/注意力→安全/算力）

### 3) 竞争与护城河（Moat）
- 网路效应类型：直接/间接/资料/协议/生态（选并解释）
- Wedge（最早必用场景，仅 1 条）：谁在什么场景下非用不可？
- 可替代性：换团队/换协议能否同样做？差异在哪？
- 失败模式：最可能死于什么（技术/治理/监管/分发/信任/激励）
  - 机制补充（可选但推荐写 1–2 条）：参与者与激励矩阵 / 攻击向量与成本收益 / 谢林点为何是它而不是竞品

### 4) 盡頭推演（Endgame Map：先 Catastrophic）
对每条路必写：终局状态 + 形成机制 + 早期信号 + 退出条件
- Catastrophic：不可逆出局（安全/共识/法律合规/声誉/治理崩）
- Bear：被替代/需求不足/信任崩
- Base：小众但稳定的有用工具
- Bull：新基础设施/新货币/新垄断平台
（可选但推荐）两问校验：  
- “本来可以发生却为什么没发生？”  
- “已堪堪发生却退回去复归不会发生？”

### 5) 非对称性评分（0–5）
逐项打分 + 一句理由：
- 尾部上行（Tail Upside）
- 生存力（Survivability）
- 早期可验证性（Time-to-Signal）
- 扩散/分发（Distribution）
- 监管/对手风险（Regulatory/Adversarial）
- 反身性（Reflexivity）
- 叙事共鸣（Narrative Resonance）
- 合法性可持续（Legitimacy Durability）
输出总评：A/B/C/D

（必写）七层共振检查（Yes/No + 一句理由，越短越好）：
- 数学/物理层（机制是否稳固）
- 技术层（可行且难复制）
- 博弈论层（激励自洽/谢林点）
- 经济学层（价值创造/供需/网络效应）
- 社会动力学层（传播/部落/反脆弱）
- 意识形态层（信念承载）
- 时代精神层（社会情绪/合法性裂缝）

共振结论：对齐层数 → 机会级别（仅作筛选，不承诺收益）：
- 7 层：更可能“改变世界级”
- 5–6 层：更可能“改变人生级”
- 3–4 层：更可能“不错的投资/产品”
- 1–2 层：更可能“有趣但不非线性”

### 6) 决策工程：探针 + 分段下注（反证优先）
- 先列 3 条最短反证路径（如何最快证明它错）
- 再列 3 条支持路径
- 只保留 1 条最短路探针（48h/7d/30d 三选一）：行动/可观测信号/退出阈值（写死）
- 分段策略（仅 Gate 3 可写）：
  - Buy Option：投入上限、时间盒、验证信号
  - Add：20%→50%→80% 升级条件（写死）
  - Exit：立刻撤退信号
  - Restart：只有哪些新证据出现才允许重启

### 7) 证据表（Evidence Table：5–10 条）
- 主张 | 文内短引句/可观测事实 | 判断（支持/反对/不确定）| 可验证方式（下一步取证）

### 8) 上帝视角洞察（必输）
#### 8.1 盲点揭示（皮肉）
- 3 个常见误区：必须落到“可观察的行为/叙事模式” + 解药（信号/探针/闸门）
#### 8.2 通关秘籍（最高效掌握路径）
- 最少步骤掌握骨架：每步必须交付产物（表/指标/访谈问题/仪表板/小作品）
#### 8.3 一句话本质
- “用【Primitive + (Belief Hook) + Flywheel + Moat】去求解【根本问题】。”

### 9) 对比与估值（必输：100x vs 1000x + 天花板）
#### 9.1 类比对照（Comparables，至少 2–3 个）
- 相同骨架（Primitive/Wedge/Flywheel/Moat/主要摩擦）+ 关键差异 + 类比最可能误导点
#### 9.2 市值估计（Bull/Base/Bear/Cat）
- 先给当前基数（token：流通市值/FDV/供给曲线；股权：估值/收入/增速）——必须 web search 核验并标注日期
- 每个情景至少两种方法交叉验证：TAM/SAM/SOM；价值捕获；网路效应溢价
- 输出数字区间（低/中/高）+ 关键假设 + 反证条件
#### 9.3 100x vs 1000x 判断（Growth Multiple）
- 计算：100x/1000x 目标市值（基于当前市值）
- 判断哪一档“物理上可行”：需要成为全球底座/行业标准/小众工具？
- 若数据不足：降级 Gate 2，只列必补资料清单

### 10) 发展路线图像（必须像图）
- 输出 ASCII/文本路线图：Phase0→1→2→3→4（每阶段：里程碑/依赖/可观测信号/失效信号/Gate 决策）
- 两条并行轨道：
  - A) 技术/产品/经济轨（PMF/分发/价值捕获/安全）
  - B) 合法性/社会轨（部落→攻击存活→符号仪式→跨部落→制度化叙事）

### 11) 认知回写（长期迭代）
- 新增假设（写入 `meta/hypotheses_registry.md`）
- 被削弱/被推翻的假设
- 新增清单项（写入 `checklists/` 或 `frameworks/`）
- 下一步要追的 3 类信息源/指标（类型即可）
- 机会图谱（可选但推荐）：当前最值得跟踪的 Top 3–5（按共振层数/可验证性排序）
- 最终一句话（可贴决策日志）：【候选】—【A/B/C/D】—【最短探针】—【停机点】—【下次回看日期】

## Hard Forcing Functions（不满足就不得 Gate 3）

每次分析必输 7 个“硬件件”（各仅 1 条）：
1) Killer Assumption（致命假设）
2) First Tripwire（最早坏信号）
3) Catastrophic Bound（不可跨红线）
4) Wedge（最早必用场景）
5) Flywheel（自增机制）
6) Death Spiral（死亡螺旋）
7) Belief Hook（信念钩子）

若缺任意一条：不得 Gate 3，只能 Gate 2（信息期权）。

## 与本项目现有体系的对接（必须）

- **猜想**：新增/更新 `choice-handbook/meta/hypotheses_registry.md`（可证伪陈述 + 证据 + 反证 + 最小验证）。  
- **决策**：若达到 80% 且允许 Gate 3 → 写入 `choice-handbook/meta/decisions_log.md`（退出/重启阈值化）。  
- **蒸馏层**：把最终可用结论压进 `choice-handbook/distilled/`（给人看的高密度版本）。
