# 05 推理工具箱（聚合版）：材料→结构→推演→行动

> 目标：把你给的一篇材料（文章/访谈/帖子/论文）快速变成**可用的推理结构**，并落到**可执行动作**。

> 这是 `frameworks/05-07` 的聚合精简版：只保留对思维推理真正有用的步骤与输出格式。

## A. 材料处理（把“输入”变成可用资产）

### A1) 分型（Type）
给材料贴标签（可多选）：采访｜论文/报告｜综述｜观点文｜教程｜讨论串｜纪要。

### A2) 可信度四格（Credibility）
- 第一手 vs 二手
- 可追溯性（数据/出处/可复现）
- 利益相关（PR/卖课/站队）
- 盲区（成本/分发/监管/组织/时间）

### A3) 资产抽取（Asset Extraction）
每类 3–7 条即可：

- 机制（因果链/互动算子）
- 结论（可证伪陈述）
- 反例/失败模式（尽头线索）
- 可验证预测（未来信号）

## B. 降维解剖（把一个领域压成“可计算骨架”）

当材料涉及一个领域/学科时，用四步压缩到可推演：
1. **事实/约束/资源/风险**（去修辞）
2. **目标函数 $\\mathcal{O}$**：最大化什么？决策变量是什么？约束是什么？

### B1) Loss Function Lens（通用决策 + ML/技术）

把“目标”写成损失/约束，不是为了数学炫技，而是为了避免：**说得很对但做不动**、或 **指标变好但现实变差**。

#### 通用决策模板（先写清楚你在惩罚什么）

用一个总损失做对齐（符号只用来帮助你写清楚结构）：

$$
L = \\text{Cost} + \\lambda\\cdot\\text{Risk} + \\mu\\cdot\\text{Regret} + \\nu\\cdot\\text{Complexity}
$$

- **Cost**：时间/金钱/精力/关系成本（你确定会付出的）
- **Risk**：尾部风险/不可逆代价（你最怕的“尽头”）
- **Regret**：后悔项（错过窗口/错过复利/错过身份与关系资产）
- **Complexity**：复杂度（越复杂越不可持续：协调成本、上下文切换、维护成本）

你需要输出 5 个字段（写成句子即可）：
- **主损失项**：我最在乎减少的是什么？（例如：现金流风险、健康透支、关系撕裂）
- **硬约束**：$g(x)\\le 0$ 的东西是什么？（例如：每周投入上限、最大回撤、家庭边界）
- **软约束/权重**：$\\lambda,\\mu,\\nu$ 大概谁最大？为什么？（阈值/阶段/偏好）
- **Proxy（代理指标）**：我用什么指标代表成功？它可能哪里会被 Goodhart？
- **正则化**：我用什么规则惩罚“不可持续”？（例如：节律/睡眠/复杂度上限）

#### ML/技术补充（把工程现实写进损失）

常见坑位与检查：
- **loss-metric mismatch**：训练损失下降，但线上/业务指标没变好（或变差）
- **surrogate loss 偏差**：为了可优化而换了代理损失，导致学到“歪解”
- **regularization**：把稳定性/泛化写进损失（否则只会过拟合短期指标）
- **约束优化**：安全/成本/延迟是硬约束还是软惩罚？如果是硬约束，别用“愿望式”损失项
- **多目标（Pareto）**：当目标冲突时，不要强行加权糊成一个数；先画 Pareto，再定权重与阈值

#### 3 个“你写错 loss/权重”的可观测信号

- **指标上升但体验变差**：说明 Proxy 失真（Goodhart）或权重错
- **短期很好、中期崩盘**：说明 Risk/Complexity 惩罚太弱（不可持续）
- **一直在优化但不敢行动**：说明你把硬约束写成了软惩罚（或把恐惧写进了无限大的 Risk）

3. **变量（≤10）+ 算子（≤5）**：状态/行动/观测/更新/成本/风险
4. **阶段切换信号（3 个）**：哪些可观测指标变了就意味着“范式切换”

## C. 比较思考（发展史 × 情景 × 信号）

当你要做趋势/路线/策略判断时：
1. **找祖先**：找 1–2 个“结构相似”的旧事物（机制相似，不是表象相似）
2. **写阶段**（3–5 段）：萌芽→扩散→挤压→平台化→常态化（不必齐全）
3. **做映射**（5–8 变量）：供给｜分发｜成本｜生态｜监管｜用户心智｜商业模式｜竞争格局
4. **写未来**：2–3 情景（保守/基准/激进）+ 每个 3 个观察信号
5. **反类比**：写清楚“哪里不像”，否则类比会变成幻觉

## C2. 历史代入（Era Lens）× 未来回看卫生（Hindsight Hygiene）× 确定性阶梯（Certainty Ladder）

> 目标：让内容更有“年代镜头感”，同时更接近确定性（靠机制/约束/信号，而不是靠文采）。

### C2.1 年代镜头（当时的人看到什么）

写 5 个字段（越具体越好）：

- **时代约束**：当时最硬的约束是什么？（技术/制度/资本/文化）
- **当时可见信号**：当时能观测到的指标/事件是什么？
- **当时主流信念**：当时的人“以为会发生什么”？为什么合理？
- **当时盲区**：当时看不到/不愿看的是什么？（激励错配、成本结构、监管、组织吞吐）
- **当时下注方式**：如果你是当时的人，你会怎么买期权/怎么做分散？

### C2.2 分叉点（当时可选的路径与代价）

至少写 2–3 条路径，每条都要有“代价与风险”：

- 路径 A：看起来最合理的主线（通常也是最拥挤的）
- 路径 B：备胎/边缘路线（通常赔率更高，但兑现更慢）
- 路径 C：反主流/反事实（通常最难，但能暴露关键机制）

### C2.3 未来回看卫生：禁止把结论伪装成当时可知

- 把信息分层写清：**当时可知** vs **事后才知**（不要混写）
- 不允许“必然会发生”：只能写 **更可能**，并给出观测信号
- 推演必须能被未来证伪：写清楚“如果没发生，最可能是哪条假设错了”

### C2.4 确定性阶梯（20% / 50% / 80%）

把未来拆成三层，不求完美，只求可更新：

- **80%（高确定）**：来自硬约束/不变量（成本曲线、物理/制度约束、激励结构）
- **50%（中确定）**：来自机制+多信号一致（但仍可能被黑天鹅改写）
- **20%（低确定）**：来自叙事与押注（需要用期权/小赌注换信息）

每层都写：**押什么机制** → **用什么信号更新** → **触发什么动作（加仓/减仓/退出/重启）**。
