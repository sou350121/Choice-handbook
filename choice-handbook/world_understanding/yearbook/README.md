# yearbook/（年度复盘：2000–2025，长版 / 06 思维）

> 作用：把每一年写成“可证伪 / 可退出 / 可复盘”的决策工程参照系。  
> 写法：显式使用 `frameworks/06_time_travel_endgame_simulator.md` 的思维：T0-Now / T0-Analog + roadmap 映射 / Endgame / 探针 / 停机点。

## 怎么用（作为决策参照系）

- 当你在做一个重大选择（职业/迁移/下注/创业/学习路径），选 2–3 个“结构相似年份”做类比（T0-Analog）。  
- 用当年的“最短反证路径”与“First Tripwire”来设你的停机点。  
- 把当年的“下一阶段最小里程碑”翻译成你现在的 7d/30d 探针。

## 生死透镜（First-Principles）：为什么有的系统活下来，有的消失？

> 用法：每次写任一年度复盘，强制做 3 步：  
> 1) 选 2–3 个最像的「成功/失败」对照案例；2) 用下面“决策轴”逐条映射；3) 输出 3 个 Tripwire + 1 个最短探针。

### A) 关键“决定轴”（不是观点，是可执行的 trade-off）

1) **流动性与期限错配**：是否把短钱（可随时挤兑/赎回）拿去做长资产/长项目？是否准备了“挤兑场景”的现金与融资备份？  
2) **杠杆与反身性控制**：是否允许高杠杆叠加“价格-抵押-清算”反馈回路？是否有硬阈值与去杠杆机制？  
3) **治理与内控（可审计/可回滚/能关机）**：是否把关键风险点做成制度化护栏（审计、权限、隔离、审批、kill-switch）？  
4) **信任边界设计（对手方/托管/隔离）**：客户资产与自营是否隔离？关键对手方是否过度集中？是否存在“单点背叛即全盘崩”的结构？  
5) **合法性与监管路径**：是否把监管当作“生存性约束”而不是成本？是否选择了可持续的合规路线、披露与可解释性？  
6) **分发与平台吞并风险**：增长是否依赖单一平台分发？是否能在平台策略变化后存活？是否具备“被下沉也能活”的差异化？  
7) **安全模型（攻击者视角）**：是否把最强攻击者当默认对手？是否有最小权限、隔离、监控、快速修复与赏金机制？  
8) **单位经济与现金流韧性**：增长是否建立在补贴/低利率/高风险偏好上？当环境逆转时是否仍能自我供血？  
9) **运营与执行（把复杂性关进流程）**：能否把高复杂系统（交易/风控/供应链/上线）做成可重复流程与应急预案？  
10) **叙事纪律（防 Goodhart）**：是否把“指标/估值/排名”当成目标，反过来扭曲了真实价值与风险管理？

### B) 最小输出协议（写任何一年都要交付）

- **3 个 Tripwire（最早坏信号）**：必须可观测、可阈值化。  
- **1 个最短探针（48h/7d/30d）**：用最小成本去验证最危险假设。  
- **1 个“看似重要但其实是噪声”的信号**：强迫自己不被故事带走。

> 06 强力版（推荐）：把上面三件事进一步压成 `frameworks/06_time_travel_endgame_simulator.md` 的 **Hard Forcing Functions**（7 件套）：  
> Killer Assumption / First Tripwire / Catastrophic Bound / Wedge / Flywheel / Death Spiral / Belief Hook。  
> 这 7 个“硬件件”就是你要找的那几个决定：它们定义了系统如何活、如何死。

### 1) Crypto / DeFi / Security（含稳定币）

#### 1.1 Bitcoin（成功） vs OneCoin（失败）
- **Outcome**：Bitcoin 作为去中心化资产长期存活；OneCoin 作为中心化诈骗迅速崩塌。  
- **Key decisions**：去中心化共识与可验证账本（无单点背叛）/ 保守升级降低攻击面 / 不靠“承诺收益”驱动需求。  
- **Tripwires**：不可验证链上数据/中心化密钥与组织控制/“保证收益”叙事。  
- **Noise**：短期涨跌与营销热度。

#### 1.2 Binance（成功） vs FTX（失败）
- **Outcome**：Binance 作为交易与流动性基础设施存活；FTX 在挤兑中因治理与负债端崩塌而归零。  
- **Key decisions**：客户资产与自营边界是否隔离 / 风控与流动性纪律是否硬约束 / 合规与披露是否当作长期生存性。  
- **Tripwires**：审计缺位/资产负债表不透明/关联方交易频繁/冻结提币。  
- **Noise**：明星背书、融资额、广告曝光。

#### 1.3 Coinbase（成功） vs Mt. Gox（失败）
- **Outcome**：Coinbase 在监管与合规框架下长期运营；Mt. Gox 因安全与治理失败出局。  
- **Key decisions**：把安全与托管当核心产品（隔离/审计/流程）/ 合规路线选择 / 事故响应能力。  
- **Tripwires**：频繁安全事件、冷/热钱包管理混乱、内部权限过大。  
- **Noise**：交易量冲高与牛市红利。

#### 1.4 Tether（成功） vs UST-Luna（失败）
- **Outcome**：Tether 作为中心化稳定币存活；UST 算法稳定币在死亡螺旋中崩溃。  
- **Key decisions**：稳定机制是否可承受大额赎回 / 是否依赖补贴与信念维持锚定 / 是否存在反身性清算链。  
- **Tripwires**：高 APY 拉存款/抵押质量不可验证/脱锚后无恢复机制。  
- **Noise**：TVL、短期生态扩张速度。

#### 1.5 Ethereum（成功） vs EOS（失败/边缘化）
- **Outcome**：Ethereum 成为通用智能合约平台；EOS 在治理/生态与需求端被边缘化。  
- **Key decisions**：生态与开发者工具链优先 / 升级治理能演化但不撕裂 / 可组合安全优先于 TPS 叙事。  
- **Tripwires**：开发者留存下滑/生态应用不生长/治理被捕获。  
- **Noise**：TPS 宣传、基金会营销。

#### 1.6 MakerDAO/DAI（相对成功） vs Celsius（失败）
- **Outcome**：DAI 以风控与清算机制穿越多轮波动；Celsius 以高收益负债端在挤兑中破产。  
- **Key decisions**：是否可观测、可关机（参数/限额/清算）/ 是否期限错配（短债吸长险）/ 风险披露是否透明。  
- **Tripwires**：高收益承诺但资产端不透明/冻结提币/单一策略过度集中。  
- **Noise**：短期收益率与用户数。

#### 1.7 Aave（成功） vs Anchor（失败/不可持续）
- **Outcome**：Aave 作为风险参数化的借贷协议长期存活；Anchor 以补贴利率驱动的模型在退潮时失效。  
- **Key decisions**：利率是否内生（供需与风险）/ 清算与风险参数是否可调 / 是否依赖外部补贴维持增长。  
- **Tripwires**：收益率长期高于可持续水平/补贴池消耗/挤兑时清算失败。  
- **Noise**：短期 TVL 与“国债级收益”叙事。

### 2) Banking / Credit Cycles（银行/信用周期）

#### 2.1 JPMorgan（成功） vs Lehman Brothers（失败）
- **Outcome**：JPM 在危机中存活并扩张；Lehman 因杠杆与融资结构在流动性断裂中破产。
- **Key decisions**：
  - 流动性缓冲与资金来源分散（不把生死押在短期批发融资）
  - 杠杆上限与资产质量纪律（尤其在繁荣末端）
  - 风险治理与压力测试是否真实执行（不是形式）
- **Tripwires**：repo/短融依赖上升、杠杆飙升、资产难以估值（Level 3）膨胀。  
- **Noise**：短期 ROE、投行业务排名、繁荣期的“风险被消灭”叙事。

#### 2.2 Goldman Sachs（成功） vs Bear Stearns（失败）
- **Outcome**：Goldman 通过更强风险与资金治理存活；Bear Stearns 在流动性与信心崩溃中被吞并。  
- **Key decisions**：流动性与融资期限结构 / 风险敞口集中度 / 压力测试与应急融资预案。  
- **Tripwires**：融资成本跳升、对手方收缩授信、资产负债表不透明。  
- **Noise**：繁荣期的高收益与“市场永远流动”幻觉。

#### 2.3 SVB（失败） vs Wells Fargo（相对稳健）
- **Outcome**：SVB 因存款集中+久期错配在挤兑中倒下；零售存款更分散者更耐久。
- **Key decisions**：
  - 负债端是否集中（单一客群、同一社群同步行为）
  - 利率风险对冲是否充分（久期管理）
  - 风险沟通与披露节奏（避免触发自我实现的挤兑）
- **Tripwires**：未对冲久期、存款高度同质化集中、社交媒体加速提款。  
- **Noise**：上一个周期的增长故事、短期利润、监管宽松幻觉。

#### 2.4 LTCM（失败） vs Renaissance Technologies（成功）
- **Outcome**：LTCM 在杠杆与尾部风险中崩溃；Renaissance 以风险控制与研究纪律长期存活。  
- **Key decisions**：杠杆上限与尾部暴露 / 头寸拥挤与流动性假设 / 模型失效时的关机机制。  
- **Tripwires**：相关性趋同、保证金上升、无法减仓（流动性消失）。  
- **Noise**：长期低波动下的“模型无敌”叙事。

### 3) Consumer Platforms（消费互联网/平台）

#### 3.1 Apple（成功） vs Nokia（失败）
- **Outcome**：Apple 通过生态与体验闭环成为平台；Nokia 在平台转移中被边缘化。
- **Key decisions**：
  - 选择“平台+生态+开发者分发”而非单点硬件优势
  - OS/应用分发生态的控制与商业化
  - 对范式变化（触屏+应用）的下注与组织执行力
- **Tripwires**：平台迁移信号出现但组织仍以旧 KPI 运转；开发者生态不增长。  
- **Noise**：功能机出货量、供应链规模、短期利润。

#### 3.2 Amazon（成功） vs Pets.com（失败）
- **Outcome**：Amazon 通过规模化履约与平台飞轮存活；Pets.com 在单位经济与资本退潮中失败。
- **Key decisions**：
  - 单位经济与现金流纪律（愿意慢，但能自我供血）
  - 物流/履约作为护城河（把成本变成能力）
  - 平台化（第三方卖家/Prime）形成飞轮
- **Tripwires**：毛利结构不支持履约成本；获客靠补贴；资本一停就断粮。  
- **Noise**：品牌声量、广告、GMV 冲高。
#### 3.3 Google Search（成功） vs Yahoo（失败/边缘化）
- **Outcome**：Google 通过产品质量+分发成为默认搜索入口；Yahoo 在入口迁移中衰落。  
- **Key decisions**：抓住“默认入口”与产品体验 / 广告商业化与用户体验的权衡 / 组织是否以产品为中心。  
- **Tripwires**：入口被替代（搜索→移动→社交）、核心产品长期停滞。  
- **Noise**：门户流量与广告收入峰值。

#### 3.4 Facebook/Meta（成功） vs MySpace（失败）
- **Outcome**：Facebook 通过产品与增长工程存活；MySpace 在产品与组织失速中衰落。  
- **Key decisions**：产品迭代节律 / 反垃圾与信任治理 / 分发与平台策略（移动化转身）。  
- **Tripwires**：用户体验被广告/垃圾内容摧毁；移动转型迟缓。  
- **Noise**：一时的访问量排名与收购溢价。

#### 3.5 TikTok（成功） vs Vine（失败）
- **Outcome**：TikTok 以推荐系统+创作者生态形成飞轮；Vine 缺商业化与生态治理被关闭。  
- **Key decisions**：创作者激励与分发机制 / 商业化节奏 / 内容治理与合规。  
- **Tripwires**：创作者流失、内容生态贫瘠、商业化无法支撑成本。  
- **Noise**：短期热梗与下载量冲高。

### 4) Cloud / SaaS / Open Source Commercialization

#### 4.1 Microsoft（成功） vs Yahoo（失败/边缘化）
- **Outcome**：MSFT 在平台周期中多次转身（Windows→Office→Cloud）；Yahoo 在分发与产品转折中衰落。
- **Key decisions**：
  - 续命的核心：抓住新的系统级分发入口（OS→生产力→云）
  - 组织能否从广告媒体叙事转到产品与平台
  - 资本配置与并购是否服务于平台重心
- **Tripwires**：核心入口被替代（搜索/移动/社交）；产品执行长期不达标。  
- **Noise**：短期流量、门户广告收入高峰。

#### 4.2 AWS（成功） vs Joyent（失败/边缘化）
- **Outcome**：AWS 通过规模化云平台成为默认；Joyent 等早期玩家在分发与规模上失速。  
- **Key decisions**：资本开支与规模化运营 / 生态与开发者入口 / 价格-性能-可靠性的长期兑现。  
- **Tripwires**：客户与生态迁移、单位成本无法下降、服务可靠性事件。  
- **Noise**：早期口碑与技术优越感。

#### 4.3 Salesforce（成功） vs Siebel（失败/被并购）
- **Outcome**：Salesforce 把 SaaS 模式做成平台；Siebel 作为传统软件在模式切换中被并购。  
- **Key decisions**：定价与交付模式（订阅/云）/ 生态与集成能力 / 客户迁移成本设计。  
- **Tripwires**：续费率下滑、实施成本过高、平台化失败。  
- **Noise**：大单签约与一次性授权收入。

#### 4.4 Red Hat（成功） vs OpenCore/许可证反复横跳（失败）
- **Outcome**：Red Hat 通过“开源+企业支持”建立可持续商业；开源商业化若破坏社区信任易被分叉与弃用。  
- **Key decisions**：许可证与贡献机制是否允许生态繁荣 / 商业化是否与社区共赢 / 治理是否避免单点控制。  
- **Tripwires**：许可证突然收紧、社区贡献断崖、分叉不回流。  
- **Noise**：短期 star 与营销式“开源宣言”。

### 5) Hardware / Semis / Supply-chain Cycles

#### 5.1 NVIDIA（成功） vs Intel（相对失速）
- **Outcome**：NVDA 抓住加速计算与生态；Intel 在制程/生态节奏上失速（阶段性）。
- **Key decisions**：
  - 押注“生态与软件栈”而不是只押硬件性能
  - 资本开支与工艺路线的节奏治理（不被单一路线锁死）
  - 与下游系统集成的协同（平台化）
- **Tripwires**：制程路线持续延误、生态迁移发生、关键客户转向。  
- **Noise**：单代产品跑分、短期价格战。

#### 5.2 供应链：Toyota（成功） vs “极致 JIT 无冗余体系”（失败）
- **Outcome**：在冲击期（疫情/地缘）冗余与可替代性决定生存；极致精益系统更脆弱。
- **Key decisions**：
  - 是否为关键零部件建立二供与库存策略（冗余=退出权）
  - 风险可视化：供应链地图与预警机制
  - 关键产能的锁定与替代路线
- **Tripwires**：单一供应商单点、地缘/封锁信号出现但无备份、交期持续拉长。  
- **Noise**：平时的成本优化 KPI、库存周转率崇拜。
#### 5.3 TSMC（成功） vs GlobalFoundries（退出先进制程）
- **Outcome**：TSMC 以制程与客户信任成为事实标准；GF 选择退出先进制程赛道。  
- **Key decisions**：资本开支与良率爬坡能力 / 客户协同与生态锁定 / 路线选择（聚焦 vs 硬拼）。  
- **Tripwires**：良率长期不达标、客户流失、资本开支无法持续。  
- **Noise**：单代节点宣传与短期订单波动。

#### 5.4 Boeing（失败/受挫） vs Airbus（相对成功）
- **Outcome**：Airbus 在质量与交付上相对更稳；Boeing 因治理/质量事故遭遇长期信任与监管冲击。  
- **Key decisions**：安全与质量是否压过成本与进度 / 供应链与认证治理 / 事故后的透明与修复速度。  
- **Tripwires**：事故频发、监管收紧、交付中断。  
- **Noise**：订单口径与短期财务优化。

#### 5.5 ASML（成功） vs Nikon（相对失败：失去关键代际）
- **Outcome**：ASML 通过 EUV 把自己变成“不可替代的设备层标准”；Nikon 在关键代际上失速。  
- **Key decisions**：押注长周期高资本开支研发 / 与产业链共研（客户协同锁定）/ 把良率与交付做成工程系统。  
- **Tripwires**：关键技术路线迟迟不兑现、客户转向、研发投入无法持续。  
- **Noise**：短期订单波动与单代参数宣传。

#### 5.6 ARM（成功） vs MIPS（失败/边缘化）
- **Outcome**：ARM 以授权生态成为事实 ISA；MIPS 在生态与分发中被边缘化。  
- **Key decisions**：生态优先（工具链/合作伙伴/授权模式）/ 兼容性与迁移成本 / 选择“做标准”而非单点产品。  
- **Tripwires**：开发者与 OEM 不迁移、工具链缺失、生态伙伴流失。  
- **Noise**：单次跑分、少量大客户的短期订单。

#### 5.7 Apple Silicon（成功） vs Intel Mac 时代（相对失速）
- **Outcome**：Apple 通过自研 SoC 把性能/功耗/体验做成闭环优势；Intel 在移动与功耗曲线下相对被动。  
- **Key decisions**：垂直整合（硬件+OS+工具链）/ 性能-功耗-成本的系统优化 / 路线下注与执行节律。  
- **Tripwires**：功耗曲线被反复击穿、制程路线延误、体验指标不再领先。  
- **Noise**：单点 benchmark 与短期价格战。

#### 5.8 Sony（成功） vs Kodak（失败）
- **Outcome**：Sony 在成像与消费电子周期中多次转型存活；Kodak 错过数字化范式而衰落。  
- **Key decisions**：是否自我颠覆旧利润池 / 押注新技术并建立新分发入口 / 组织 KPI 是否能从“胶片利润”迁移到“数字体验”。  
- **Tripwires**：新技术渗透率拐点出现但组织仍守旧 KPI、核心产品线现金流快速下滑。  
- **Noise**：旧业务短期仍赚钱带来的侥幸。

### 6) AI / Models / Agents / Platform Capture

#### 6.1 OpenAI（成功） vs Inflection AI（失速/转向）
- **Outcome**：少数能形成系统级能力与分发的玩家存活；薄工具在模型同质化与平台下沉中被挤压。
- **Key decisions**：
  - 把能力沉到系统记录/工作流（SoR）与结果闭环数据
  - 分发与平台协同（而不是只靠功能）
  - 安全与合规护栏（避免 catastrophic）
- **Tripwires**：留存与定价权在平台更新后塌陷；无独家数据/无结果闭环；合规事故。  
- **Noise**：demo 演示、短期投放带来的“用户数”。

#### 6.2 NVIDIA CUDA（成功） vs “无生态的 AI 芯片挑战者”（失败）
- **Outcome**：CUDA 把硬件变平台；无生态挑战者即使有芯片也难撼动。  
- **Key decisions**：软件栈与开发者生态优先 / 兼容性与迁移成本设计 / 与系统集成商协同。  
- **Tripwires**：开发者不迁移、工具链不成熟、性能优势无法兑现到端到端。  
- **Noise**：单点跑分、单一 benchmark 宣传。

#### 6.3 Agent 产品：可控执行（成功） vs 越权/事故（失败）
- **Outcome**：能建立最小权限/审计/回滚的工作流系统存活；事故频发者迅速失信。  
- **Key decisions**：默认最小权限+二次确认 / 审计与可回滚 / 把提示注入当默认对手。  
- **Tripwires**：不可解释修改/误删、越权调用工具、日志缺失。  
- **Noise**：多智能体热闹、短期效率宣称。

#### 6.4 Hugging Face（成功） vs “只做模型发布、缺生态闭环的平台”（失败/被挤压）
- **Outcome**：HF 通过模型/数据/评测/社区把自己变成默认入口；缺生态闭环的平台在同质化下被边缘化。  
- **Key decisions**：把“分发入口”做成基础设施（托管、权限、版本、卡片、评测）/ 社区治理与贡献机制 / 与企业需求（合规/私有化）兼容。  
- **Tripwires**：社区贡献断崖、平台无法承载企业合规与私有化、被更底层平台（云/OS/IDE）吞并。  
- **Noise**：短期 star 暴涨与营销式发布会。

#### 6.5 DeepMind AlphaFold（成功） vs IBM Watson Health（失败/收缩）
- **Outcome**：AlphaFold 以可验证的科学产出与工具化落地形成长期影响；Watson Health 在承诺与交付落差中收缩。  
- **Key decisions**：选择“可验证任务”（公开基准/可复现评测）/ 把能力产品化为工具链 / 医疗场景的合规与真实数据闭环。  
- **Tripwires**：对外承诺无法在可验证指标兑现、合规/数据闭环拿不到、落地成本高于收益。  
- **Noise**：媒体叙事、概念 demo、模糊 KPI。

#### 6.6 GitHub Copilot（成功） vs “无分发入口的代码助手”（失败/边缘化）
- **Outcome**：Copilot 依托 IDE/代码托管分发形成默认入口；缺分发入口者即使能力相近也难形成留存。  
- **Key decisions**：选择“工作流入口”（IDE/Repo）/ 把质量指标做成可回归（接受率、节省时间、事故率）/ 权限与安全护栏。  
- **Tripwires**：平台原生功能覆盖、留存与愿付价崩塌、安全事故或许可证/隐私争议。  
- **Noise**：短期试用用户数与社媒热度。

#### 6.7 图像生成：Midjourney（成功） vs “可复制的工具站”（失败/被卷死）
- **Outcome**：Midjourney 以社区与品味+迭代形成黏性；可复制工具站在模型同质化与渠道成本上升中被卷死。  
- **Key decisions**：建立社区与创作反馈闭环 / 迭代速度与产品体验 / 合规与版权风险治理。  
- **Tripwires**：渠道获客成本上升、模型能力同质化、版权/合规风险爆雷。  
- **Noise**：一次爆款风格与短期付费转化。

### 7) Energy / Commodities / Utilities

#### 7.1 Enron（失败） vs Berkshire Hathaway Energy（成功）
- **Outcome**：Enron 在治理与风险不透明中崩溃；BHE 作为现金流公用事业长期存活。  
- **Key decisions**：透明会计与风险披露 / 把现金流当底座而非金融工程 / 治理与内控。  
- **Tripwires**：报表复杂不可解释、对手方与衍生品敞口爆炸、审计与治理预警。  
- **Noise**：创新叙事、短期估值与媒体赞美。

#### 7.2 OPEC+（成功的供给协调） vs Shale 高杠杆扩张（失败/破产潮）
- **Outcome**：供给协调者能在周期中维持定价权；高杠杆扩张在油价下行时出现破产潮。  
- **Key decisions**：是否以杠杆赌价格 / 是否有对冲与资本纪律 / 产能扩张的节奏治理。  
- **Tripwires**：对冲不足、债务到期墙、现金流为负但继续扩张。  
- **Noise**：油价单边行情与“永远高油价”叙事。

#### 7.3 NextEra Energy（成功） vs SunEdison（失败/破产）
- **Outcome**：NextEra 以资本纪律与项目运营做成可持续复利；SunEdison 以杠杆+并购扩张崩溃。  
- **Key decisions**：项目回报率与现金流优先 / 杠杆与并购节奏治理 / 资金成本上行时的停机策略。  
- **Tripwires**：资金成本上升、项目现金流覆盖不了债务、并购整合失控。  
- **Noise**：装机规模排名与“新能源永远高增长”叙事。

#### 7.4 Tesla（成功） vs Nikola（失败/欺诈/崩塌）
- **Outcome**：Tesla 通过制造与交付闭环存活并扩张；Nikola 因不可验证承诺与治理问题崩塌。  
- **Key decisions**：把关键指标从“叙事”落到“交付与毛利” / 供应链与质量体系 / 治理与披露。  
- **Tripwires**：关键技术不可验证、交付持续为 0、监管/诉讼风险上升。  
- **Noise**：预订量、概念车与媒体热度。

#### 7.5 公用事业：稳健现金流（成功） vs 高风险交易台（失败）
- **Outcome**：把公用事业当“现金流资产”的公司更抗冲击；把核心利润建立在交易/衍生品的公司更脆弱。  
- **Key decisions**：风险敞口透明化 / 对手方与保证金管理 / 把收益从交易波动迁移到可预测现金流。  
- **Tripwires**：保证金压力、对手方集中、利润来源无法解释。  
- **Noise**：短期高 ROE 与“交易能力”神话。

#### 8.2 Pfizer（成功） vs “小公司单点押注临床失败”（失败）

- **Key decisions**：管线分散与现金流底座 / 临床里程碑管理 / 合作与授权策略。  
- **Tripwires**：单一试验失败即现金流断裂、融资窗口关闭。  
- **Noise**：早期动物数据与过度解读。

#### 8.3 Semaglutide 生态（成功：Novo Nordisk 等） vs “单点减肥神药叙事”（失败）
- **Outcome**：能把疗效、安全性、供应与支付体系做成闭环的公司长期兑现；单点叙事（无供应/无支付/无长期数据）容易破裂。  
- **Key decisions**：长期临床数据与安全性 / 产能与供应链 / 支付方与处方路径（合规）。  
- **Tripwires**：安全性信号、供应断裂、支付方收紧或监管限制。  
- **Noise**：短期处方爆发与社媒热度。

#### 8.4 医疗器械：合规与质量（成功） vs 伪创新绕监管（失败）
- **Outcome**：按监管逻辑迭代的器械公司更可持续；绕监管与数据造假者在审计/事故中出局。  
- **Key decisions**：质量体系与临床证据 / 不把营销当交付 / 事故披露与召回治理。  
- **Tripwires**：不良事件上升、审计缺陷、召回与诉讼。  
- **Noise**：短期增长与渠道压货。

- **Key decisions**：合规与数据本地化策略 / 组织与资本结构（降低地缘敏感性）/ 多市场分发冗余。

#### 9.4 加密：Binance（成功/存活） vs “无合规护城河的交易所”（失败）

- **Key decisions**：KYC/AML 与风控体系 / 资产隔离与透明度 / 选择在关键辖区落地合规。  
- **Tripwires**：监管诉讼升级、资金流出加速、储备与负债不透明。  
- **Noise**：短期交易量与手续费暴利。

### 11) Financial Crises / Banking / Leverage（金融危机：杠杆×期限×对手方）

> 这组对照的核心不是“市场预测”，而是：当 **流动性枯竭 / 融资成本上行 / 对手方信任崩** 时，谁能活下来？

#### 11.1 Lehman Brothers（失败） vs JPMorgan（成功/存活）
- **Outcome**：Lehman 因融资结构与信任崩塌破产；JPM 以更强风控与流动性缓冲存活并扩张。  
- **Key decisions**：流动性覆盖与融资来源多样化 / 风险敞口透明与对手方管理 / 在繁荣期压杠杆而非追 ROE。  
- **Tripwires**：短融依赖升高、抵押品折价扩大、对手方收紧、评级与保证金压力上升。  
- **Noise**：短期盈利高、复杂结构“看起来对冲了”。

#### 11.2 Bear Stearns（失败） vs Goldman Sachs（相对成功/存活）
- **Outcome**：Bear 在挤兑中迅速失去融资；Goldman 通过更强风险管理与自救（含资本/融资调整）存活。  
- **Key decisions**：仓位与杠杆的动态降维 / 抵押品与保证金管理 / 关键时刻的融资渠道与信用背书。  
- **Tripwires**：repo 续作失败、prime brokerage 客户撤离、保证金/折价快速恶化。  
- **Noise**：单季交易收入与“风控模型先进”的叙事。

#### 11.3 AIG（失败/被救） vs Berkshire Hathaway（成功）
- **Outcome**：AIG 因衍生品与对手方链条被迫救助；Berkshire 以现金流底座与保守杠杆穿越周期。  
- **Key decisions**：不把尾部风险当“可忽略” / 保险与金融工程的边界治理 / 保持可随时出手的现金与信用。  
- **Tripwires**：保证金条款触发、对手方集中、模型假设在极端情景失效。  
- **Noise**：长期低波动带来的“风险被消灭了”幻觉。

#### 11.4 Long-Term Capital Management（失败） vs “低杠杆/可退出的套利体系”（成功）
- **Outcome**：LTCM 因高杠杆与相关性趋同而崩；低杠杆、可退出的套利更能存活。  
- **Key decisions**：把“相关性趋 1”当必然情景 / 杠杆上限与流动性约束 / 退出路径（市场深度/对手方条款）。  
- **Tripwires**：价差扩张但流动性消失、保证金提高、对手方拒绝展期。  
- **Noise**：历史回测胜率与“均值回归必然”信念。

#### 11.5 Northern Rock（失败） vs 传统存款银行（相对成功）
- **Outcome**：过度依赖批发融资的银行在挤兑中更脆弱；存款更稳定的银行相对更抗冲击。  
- **Key decisions**：资产负债期限匹配 / 融资结构（存款 vs 批发融资）/ 预案（流动性应急与沟通）。  
- **Tripwires**：短期融资占比上升、同业拆借紧张、媒体与社交信任风暴。  
- **Noise**：扩张速度与利润率。

#### 11.6 Money Market：Reserve Primary Fund（失败） vs “更保守的现金管理”（成功）
- **Outcome**：打破 1 美元净值会触发恐慌赎回；更保守的现金管理能避免“现金=不安全”的信任崩。  
- **Key decisions**：资产质量与流动性优先 / 对手方分散 / 透明披露与赎回机制。  
- **Tripwires**：高收益资产占比上升、单一发行人集中、赎回加速。  
- **Noise**：短期收益率排名。

#### 11.7 SVB（失败） vs “久期对冲与存款多样化”（成功）
- **Outcome**：SVB 因存款集中+久期管理失误在加息周期触发挤兑；更分散且对冲的机构更稳。  
- **Key decisions**：久期与利率风险对冲 / 存款客户结构多样化 / 预案（融资窗口与沟通）。  
- **Tripwires**：未实现亏损扩大、存款高度同质、社交媒体/VC 协调赎回。  
- **Noise**：会计口径利润与“长期持有就没亏”的叙事。

#### 11.8 欧债：希腊（失败/重组） vs 德国（相对成功）
- **Outcome**：无货币主权的高债国家更易被融资成本锁死；信用与制度背书更强者更稳。  
- **Key decisions**：财政纪律与税基 / 结构改革与竞争力 / 危机时的外部背书与条件交换。  
- **Tripwires**：利差持续走阔、融资窗口关闭、政治承诺与执行脱节。  
- **Noise**：短期刺激与一次性救助方案。

#### 11.9 日本“失去的十年”（相对失败） vs 更快出清的体系（相对成功）
- **Outcome**：拖延出清会导致长期低增长与资产负债表衰退；更快出清更可能恢复活力。  
- **Key decisions**：坏账处置与资本重组 / 货币与财政配合 / 结构性改革。  
- **Tripwires**：僵尸企业占比上升、银行不良不出清、信用传导失灵。  
- **Noise**：短期反弹与资产价格暂时企稳。

#### 11.10 “单点大到不能倒”金融集团（失败模式） vs 去集中/可拆分架构（成功）
- **Outcome**：复杂到无法处置的结构在危机中成本极高；可拆分与清算机制完善者更稳。  
- **Key decisions**：结构简化与可处置性（resolvability）/ 透明度与穿透式风险 / 监管沟通。  
- **Tripwires**：关键资产无法估值、内部交易/担保链条不透明、监管强制介入。  
- **Noise**：并购扩张与“协同效应”口径。

### 12) Platform Wars（平台战争：入口×生态×迁移成本）

> 平台战争的决定性变量通常不是“功能”，而是：**默认入口（distribution）**、**开发者生态**、**数据/身份系统记录（SoR）**、**迁移成本**。

#### 12.1 iOS（成功） vs Symbian（失败）
- **Outcome**：iOS 以开发者生态与体验标准化赢得智能手机平台战争；Symbian 在生态与迭代节奏中出局。  
- **Key decisions**：统一平台与 SDK / 应用分发与支付闭环 / 快速迭代与体验一致性。  
- **Tripwires**：开发者流失、应用生态不繁荣、版本碎片化。  
- **Noise**：单机型硬件参数领先。

#### 12.2 Android（成功） vs Windows Phone（失败）
- **Outcome**：Android 通过 OEM 生态与免费授权扩张；WP 因生态不足与迟疑策略失败。  
- **Key decisions**：生态扩张优先 / 兼容性与分发策略 / 与硬件伙伴的利益对齐。  
- **Tripwires**：关键应用缺失、OEM 不投入、开发者 ROI 不成立。  
- **Noise**：少数旗舰评测口碑。

#### 12.3 Chrome（成功） vs Internet Explorer（失速/边缘化）
- **Outcome**：Chrome 以更快迭代与标准推动赢得浏览器平台；IE 在安全与标准债务中失速。  
- **Key decisions**：迭代节奏与自动更新 / 标准与开发者工具 / 安全与性能工程。  
- **Tripwires**：安全事故频发、标准不兼容、开发者开始“先适配其他”。  
- **Noise**：短期市场份额口径与预装优势。

#### 12.4 Microsoft（成功：Win/Office 时代） vs Netscape（失败）
- **Outcome**：OS 分发与捆绑策略决定早期浏览器战争；Netscape 在分发与资源战中失败。  
- **Key decisions**：掌握默认入口（OS）/ 定价与捆绑策略 / 开发者工具与生态。  
- **Tripwires**：被更底层平台切断分发、开发者转向、商业模型无法续航。  
- **Noise**：早期口碑与技术领先。

#### 12.5 Facebook（成功） vs MySpace（失败）
- **Outcome**：Facebook 以更强产品体验、实名关系链与分发增长存活；MySpace 因产品债务与治理失速。  
- **Key decisions**：关系链质量与推荐机制 / 产品工程治理 / 生态与内容治理。  
- **Tripwires**：垃圾内容泛滥、留存下降、体验崩坏。  
- **Noise**：短期 PV 与广告收入。

#### 12.6 WeChat 小程序（成功） vs “孤立的轻应用平台”（失败）
- **Outcome**：依托超级入口与支付/身份，生态能自增长；无入口与支付闭环的平台难形成网络效应。  
- **Key decisions**：入口与支付绑定 / 开发者激励与治理 / 体验与审计（合规）。  
- **Tripwires**：低质生态、用户转化差、监管限制。  
- **Noise**：平台上架数量与活动口径。

#### 12.7 AWS（成功） vs “被价格战拖死的云服务商”（失败）
- **Outcome**：规模经济+生态+可靠性形成护城河；单纯价格战在资本周期逆转时易崩。  
- **Key decisions**：产品组合与可组合 API / 可靠性工程 / 资本纪律与单位经济学。  
- **Tripwires**：单位成本不下降、客户流失、可靠性事故频发。  
- **Noise**：短期促销带来的收入增长。

#### 12.8 移动支付：支付宝/微信支付（成功） vs “只做钱包不做场景”（失败）
- **Outcome**：支付成功依赖场景与网络效应；纯钱包缺场景与商户网络难形成闭环。  
- **Key decisions**：线下商户网络 / 补贴节奏 / 风控与合规。  
- **Tripwires**：欺诈上升、监管收紧、商户网络扩张停滞。  
- **Noise**：下载量与补贴拉新数据。

#### 12.9 游戏平台：Steam（成功） vs “缺社区与分发的商店”（失败/边缘化）
- **Outcome**：Steam 以分发+社区+资产沉淀形成护城河；缺社区与资产沉淀者更脆弱。  
- **Key decisions**：社区与社交关系链 / 开发者分成与工具 / 用户资产（库与账号）迁移成本。  
- **Tripwires**：开发者不投入、用户资产不可迁移、口碑与信任崩。  
- **Noise**：短期独占与补贴。

#### 12.10 开放标准（成功） vs 封闭标准（失败/被替代）
- **Outcome**：开放标准更易成为行业默认；封闭标准在生态竞争中更易被边缘化。  
- **Key decisions**：标准开放程度 / 兼容性与迁移路径 / 治理与利益分配。  
- **Tripwires**：开发者绕开、兼容层出现、生态伙伴离开。  
- **Noise**：短期授权收入与封闭“控制力”快感。

### 13) Supply Chain & Manufacturing（供应链与制造：质量×交付×冗余）

> 这组对照的核心是：当冲击发生时（疫情/地缘/原材料/融资成本），谁能保持 **交付** 与 **现金流**？

#### 13.1 SpaceX（成功） vs 传统航天承包商模式（相对失速）
- **Outcome**：SpaceX 以垂直整合+高频迭代+规模制造建立成本曲线优势；传统承包商在激励与交付节奏中更慢。  
- **Key decisions**：垂直整合与可控供应链 / 迭代速度与测试文化 / 把成本曲线当护城河。  
- **Tripwires**：迭代频率下降、质量事故无法快速修复、供应链单点卡死。  
- **Noise**：单次发射失败/成功的舆论波动。

#### 13.2 Zara（成功） vs Gap（相对失速）
- **Outcome**：Zara 以快反供应链与数据驱动补货存活；Gap 在周期与库存错配中受挫。  
- **Key decisions**：近岸产能与快反 / 库存与需求预测 / 供应链与门店协同。  
- **Tripwires**：库存周转恶化、折扣依赖上升、补货周期过长。  
- **Noise**：单季爆款与短期促销拉动。

#### 13.3 Apple（成功） vs “只做品牌、制造不可控”（失败模式）
- **Outcome**：Apple 通过供应链治理与关键环节掌控实现长期交付与毛利；制造不可控的品牌在冲击期更脆弱。  
- **Key decisions**：关键元件锁定与多源 / 质量与良率工程 / 现金预付款与产能投资（用资本换确定性）。  
- **Tripwires**：关键零部件断供、良率崩、交付延期导致渠道信任崩。  
- **Noise**：短期毛利率波动与代工价格谈判。

#### 13.4 Shipping：Maersk（成功/转型） vs Hanjin Shipping（失败/破产）
- **Outcome**：资本纪律与资产负债表决定航运周期生死；过度杠杆在下行周期破产。  
- **Key decisions**：资本开支与杠杆上限 / 长约与客户结构 / 现金流与资产处置速度。  
- **Tripwires**：运价下行+债务到期墙、融资窗口关闭、客户流失。  
- **Noise**：单边景气期的“永远高运价”叙事。

#### 13.5 制造质量：Toyota（相对成功） vs “以返工/售后掩盖质量问题”（失败）
- **Outcome**：质量体系是长期护城河；用售后掩盖质量会在规模化与舆论中爆雷。  
- **Key decisions**：质量门禁（stop-the-line）/ 供应商质量协同 / 把缺陷当系统问题而非个体问题。  
- **Tripwires**：召回频发、关键缺陷复发、供应商质量失控。  
- **Noise**：短期交付数量与成本 KPI。

#### 13.6 电池与电动车：垂直整合（成功） vs 供应链拼装（失败模式）
- **Outcome**：关键部件（电池/电驱/软件）可控者更能穿越周期；拼装模式在供给冲击中更脆弱。  
- **Key decisions**：关键部件自研或深度绑定 / 产能与原材料锁定 / 软件与热管理的系统工程。  
- **Tripwires**：原材料价格冲击、供应商断供、良率与安全事故。  
- **Noise**：预售订单与补贴期销量。

#### 13.7 半导体：IDM（成功） vs Fabless 但无代工保障（失败模式）
- **Outcome**：产能保障与路线执行决定能否交付；无产能保障在周期与地缘中更脆弱。  
- **Key decisions**：产能锁定（long-term agreements）/ 多源与备选工艺 / 产品路线与节点选择。  
- **Tripwires**：产能挤兑、关键节点被卡、客户因交期转向。  
- **Noise**：单代性能宣传与短期需求波动。

#### 13.8 “极致降本外包”（失败） vs “关键环节内生能力”（成功）
- **Outcome**：把关键能力外包会在冲击期暴露脆弱；关键环节内生（或强绑定）提升生存力。  
- **Key decisions**：识别关键路径并保留控制权 / 外包合同的激励一致性 / 冗余与备份机制。  
- **Tripwires**：外包方产能/质量崩、合同无法约束、关键交付延期。  
- **Noise**：短期毛利提升与财务报表好看。

### 14) Media / Content / Creator Platforms（媒体/内容/创作者）

> 这组对照的核心是：**分发入口**、**版权/内容成本曲线**、**用户留存与付费结构**。

#### 14.1 Netflix（成功） vs Blockbuster（失败）
- **Outcome**：Netflix 通过订阅与流媒体迁移存活；Blockbuster 在模式转移中失速。  
- **Key decisions**：订阅模型与反盗版策略 / 从 DVD 到流媒体的前置迁移 / 内容投资与全球分发。  
- **Tripwires**：内容成本超出 LTV、流失率上升、关键版权丧失。  
- **Noise**：单部爆款与短期新增。  

#### 14.2 Spotify（成功） vs Pandora（相对失速）
- **Outcome**：Spotify 以双边市场与推荐系统扩张；Pandora 在规模与授权成本上受限。  
- **Key decisions**：全球授权与分成结构 / 发现与推荐引擎 / 播客与广告变现。  
- **Tripwires**：版权费用占比长期不降、MAU 增长停滞、付费转化下降。  
- **Noise**：短期榜单与单曲热度。  

#### 14.3 NYT 订阅（成功） vs BuzzFeed（广告依赖失败/收缩）
- **Outcome**：NYT 以订阅与内容资产穿越流量波动；广告依赖型媒体在平台算法变化中收缩。  
- **Key decisions**：订阅与捆绑产品 / 编辑与调查投入 / 多元收入（游戏/播客/工具）。  
- **Tripwires**：平台流量占比过高、广告 RPM 下降、用户付费停滞。  
- **Noise**：病毒式传播与短期 PV 峰值。  

#### 14.4 Disney+（相对成功） vs Quibi（失败）
- **Outcome**：Disney+ 依靠 IP 库与分发进入规模；Quibi 因内容与分发错配快速失败。  
- **Key decisions**：IP 资产与上新节奏 / 设备与渠道覆盖 / 试用转正与捆绑策略。  
- **Tripwires**：试用后流失率高、内容成本>留存价值、下载后活跃不升。  
- **Noise**：上线首月下载量。

### 15) Mobility / Transportation / Urban Infra（交通/出行/城市基础设施）

> 这组对照的核心是：**资本强度**、**网络效应/调度能力**、**安全与合规门槛**。

#### 15.1 Tesla（成功） vs Better Place（失败）
- **Outcome**：Tesla 以垂直整合与充电网络形成壁垒；Better Place 因基础设施与商业模型失配失败。  
- **Key decisions**：电池/软件/整车一体化 / 充电网络与补能体验 / 资本开支与规模节奏。  
- **Tripwires**：电池成本曲线停滞、充电网络利用率下降、监管或安全事故冲击。  
- **Noise**：短期交付波动与单季利润。  

#### 15.2 Uber（成功） vs 传统出租车牌照体系（失速）
- **Outcome**：Uber 以动态定价与供给调度扩张；牌照体系在用户体验与供给弹性上失速。  
- **Key decisions**：动态定价保障供给 / 多城市扩张与合规策略 / 产品体验与安全体系。  
- **Tripwires**：司机供给下滑、监管禁令、单位经济学长期为负。  
- **Noise**：补贴拉动的 GMV。  

#### 15.3 Shinkansen（成功） vs Concorde（失败）
- **Outcome**：新干线以高频可靠服务成为公共基础设施；协和因成本与需求结构失败。  
- **Key decisions**：面向大众的容量与可靠性 / 运维成本控制 / 政策与安全标准。  
- **Tripwires**：上座率低于盈亏平衡、维护成本失控、补贴缩减。

- **Tripwires**：支付成功率下降、拒付率上升。  
- **Noise**：单月交易量冲高。  

---

### 29) Data / Observability / DevOps（数据/可观测性/DevOps）

#### 29.1 统一可观测平台（相对成功） vs 工具拼盘（失败模式）
- **Outcome**：统一平台提升排障效率；工具拼盘导致盲区与误判。  
- **Key decisions**：统一采集标准 / MTTR 目标 / 事故复盘机制。  
- **Tripwires**：排障时间上升、重复事故频发。  
- **Noise**：单次“救火”成功。  

#### 29.2 云数仓（相对成功） vs 本地数仓（失速）
- **Outcome**：云数仓弹性扩展与成本可控；本地数仓在负载与维护上失速。  
- **Key decisions**：弹性与成本模型 / 迁移与兼容性 / 性能治理。  
- **Tripwires**：查询排队时间上升、维护窗口频繁。  
- **Noise**：短期成本上涨或迁移舆论。  

#### 29.3 CI/CD 自动化（相对成功） vs 手工发布（失败模式）
- **Outcome**：自动化降低发布风险与频率成本；手工发布在复杂系统中失败。  
- **Key decisions**：自动化覆盖率 / 回滚能力 / 发布频率治理。  
- **Tripwires**：发布失败率上升、回滚频繁。  
- **Noise**：单次发布顺利。

### 30) Healthcare Delivery（医疗服务/照护体系）

#### 30.1 整合式医疗（相对成功） vs 碎片化收费（失败模式）
- **Outcome**：整合式照护更能控制结果与成本；碎片化体系导致重复检查与成本失控。  
- **Key decisions**：连续性照护 / 成本与结果管理 / 资源配置。  
- **Tripwires**：再入院率上升、重复检查比例增加。  
- **Noise**：短期门诊量上涨。  

#### 30.2 可持续远程医疗（相对成功） vs 补贴驱动（失败模式）
- **Outcome**：可持续模式保留高复诊与粘性；补贴驱动在退潮后塌陷。  
- **Key decisions**：复诊与粘性设计 / 医生供给保障 / 报销与合规策略。  
- **Tripwires**：复诊率下降、医生供给不足。  
- **Noise**：短期新增用户冲高。  

#### 30.3 价值医疗（相对成功） vs 量计费（失败模式）
- **Outcome**：按结果付费推动效率与健康结果；按量收费激励过度诊疗。  
- **Key decisions**：结果指标与激励 / 成本控制 / 数据与评估能力。  
- **Tripwires**：质量指标恶化、患者满意度下降。  
- **Noise**：短期手术量增长。  

---

### 31) Materials / Chemicals（材料/化工/基础化学）

#### 31.1 多源原料体系（相对成功） vs 单一原料依赖（失败模式）
- **Outcome**：多源原料更能抗断供；单一原料依赖在冲击期停产。  
- **Key decisions**：原料多源化 / 安全库存 / 供应链可替代性。  
- **Tripwires**：原料价格异常、单一供应占比上升。  
- **Noise**：短期现货价格波动。  

#### 31.2 严格安全管理（相对成功） vs 事故频发工厂（失败模式）
- **Outcome**：安全体系降低长期停产；事故频发导致监管介入与产能损失。  
- **Key decisions**：安全投入与整改闭环 / 设备维护 / 培训机制。  
- **Tripwires**：事故频次上升、整改未闭环。  
- **Noise**：短期产量反弹。  

#### 31.3 有序扩产（相对成功） vs 过度扩产（失败模式）
- **Outcome**：有序扩产维持利润；过度扩产在周期下行时亏损。  
- **Key decisions**：扩产节奏 / 供需判断 / 资本纪律。  
- **Tripwires**：库存上升、现货价持续下跌。  
- **Noise**：单季需求反弹。  

---

### 32) Mining / Metals（矿业/金属/资源）

#### 32.1 高品位矿（相对成功） vs 低品位矿（失败模式）
- **Outcome**：高品位矿现金成本低、更能抗周期；低品位矿在价格下行期停产。  
- **Key decisions**：品位维护与成本控制 / 能源效率 / 现金流弹性。  
- **Tripwires**：现金成本接近现货价、品位下滑。  
- **Noise**：短期价格反弹。  

#### 32.2 分阶段开发（相对成功） vs 大跃进扩产（失败模式）
- **Outcome**：分阶段开发控制风险；大跃进扩产易超支延期。  
- **Key decisions**：分期投资 / 工期与审批治理 / 资本纪律。  
- **Tripwires**：工程延期、预算超支。  
- **Noise**：短期产能目标宣示。  

#### 32.3 合规治理（相对成功） vs 社会阻力停工（失败模式）
- **Outcome**：良好社区关系与合规可持续；冲突导致停工与许可证风险。  
- **Key decisions**：社区关系管理 / 许可合规 / 环境与复垦投入。  
- **Tripwires**：社区投诉上升、监管调查增加。  
- **Noise**：单次地方政府表态。  

---

### 33) Logistics / Trade / Ports（物流/贸易/港口）

#### 33.1 多通道物流（相对成功） vs 单通道依赖（失败模式）
- **Outcome**：多通道可快速切换避免停摆；单通道依赖在封锁/事故中崩溃。  
- **Key decisions**：多通道布局 / 应急切换能力 / 客户协同。  
- **Tripwires**：通道拥堵指数上升、替代路径不可用。  
- **Noise**：短期运价波动。  

#### 33.2 精益周转（相对成功） vs 库存堆积（失败模式）
- **Outcome**：精益周转降低资金占用；库存堆积在需求下行期现金流断裂。  
- **Key decisions**：库存与周转管理 / 采购节奏 / 融资成本控制。  
- **Tripwires**：库存周转天数上升、折扣依赖加大。  
- **Noise**：单季补库行为。  

#### 33.3 数字化港口（相对成功） vs 低效港口（失败模式）
- **Outcome**：数字化调度提升吞吐；低效港口在高峰期拥堵失控。  
- **Key decisions**：调度系统与协同 / 投资节奏 / 数据治理。  
- **Tripwires**：平均等待时间上升、积压集装箱增加。  
- **Noise**：短期天气扰动。  

---

### 34) Public Policy / State Capacity（公共治理/国家能力）

#### 34.1 稳健财政（相对成功） vs 财政透支（失败模式）
- **Outcome**：稳健财政更能穿越危机；财政透支在冲击时失灵。  
- **Key decisions**：财政纪律 / 债务管理 / 风险准备金。  
- **Tripwires**：债务率快速上升、融资成本飙升。  
- **Noise**：短期刺激与口号。  

#### 34.2 强执行力（相对成功） vs 纸面政策（失败模式）
- **Outcome**：执行力强的政策能落地；纸面政策无法转化为结果。  
- **Key decisions**：执行链条设计 / 反馈与问责 / 协同机制。  
- **Tripwires**：地方执行差异大、监督与问责缺失。  
- **Noise**：短期数据美化。  

#### 34.3 高信任社会（相对成功） vs 低信任社会（失败模式）
- **Outcome**：高信任降低治理成本；低信任导致合规成本上升。  
- **Key decisions**：信任建设 / 透明度与沟通 / 法治与公平。  
- **Tripwires**：抗议频率上升、合规率下降。  
- **Noise**：短期民调波动。
