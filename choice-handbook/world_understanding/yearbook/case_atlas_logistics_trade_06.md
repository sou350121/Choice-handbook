## 33.x 物流/贸易/港口（06 硬化案例）

### 33.1 多通道物流（相对成功） vs 单通道依赖（失败模式）
- **Outcome**：多通道可快速切换避免停摆；单通道依赖在封锁/事故中崩溃。  
- **T0 环境**：地缘与极端事件频发。  
- **Killer Assumption**：主通道永不被堵。  
- **Wedge**：事故/战争/封锁导致通道中断。  
- **Flywheel**：多通道保障↑ → 交付稳定↑ → 客户信任↑ → 订单稳。  
- **First Tripwire**：通道拥堵指数上升、替代路径不可用。  
- **Catastrophic Bound**：通道中断超过库存缓冲期。  
- **Death Spiral**：延误 → 违约/赔付 → 现金流下降 → 备选通道投入减少 → 延误更大。  
- **Noise**：短期运价波动。  
- **最短探针（48h）**：列“Top3 通道依赖度 + 替代路径可用性”，低于阈值即判单点风险。  

### 33.2 精益周转（相对成功） vs 库存堆积（失败模式）
- **Outcome**：精益周转降低资金占用；库存堆积在需求下行期现金流断裂。  
- **T0 环境**：需求波动大、融资成本上升。  
- **Killer Assumption**：库存永远能快速周转或折价出清。  
- **Wedge**：需求下滑与运价上升。  
- **Flywheel**：周转快 → 现金流稳 → 议价力↑ → 周转更快。  
- **First Tripwire**：库存周转天数上升、折扣依赖加大。  
- **Catastrophic Bound**：库存占用超过融资能力。  
- **Death Spiral**：库存↑ → 现金流↓ → 采购缩减 → 交付差 → 订单更少。  
- **Noise**：单季补库行为。  
- **最短探针（7d）**：跟踪“库存周转天数 + 融资成本”，超过阈值即判现金流风险。  

### 33.3 数字化港口（相对成功） vs 低效港口（失败模式）
- **Outcome**：数字化调度提升吞吐；低效港口在高峰期拥堵失控。  
- **T0 环境**：全球贸易高峰与船期密集。  
- **Killer Assumption**：传统人工作业足以应对高峰。  
- **Wedge**：拥堵与排队导致周转失控。  
- **Flywheel**：调度效率↑ → 吞吐↑ → 收入↑ → 再投资↑。  
- **First Tripwire**：平均等待时间上升、积压集装箱增加。  
- **Catastrophic Bound**：港口拥堵导致大面积改港与合约违约。  
- **Death Spiral**：拥堵 → 船期延误 → 货主转移 → 收入下降 → 投资不足 → 更拥堵。  
- **Noise**：短期天气扰动。  
- **最短探针（30d）**：监测“平均等待时间 + 吞吐率”，低于阈值即判港口失序。
