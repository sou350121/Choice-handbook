## 28.x 支付/金融基础设施（06 硬化案例）

### 28.1 开放式支付网络（相对成功） vs 闭环小网络（失败模式）
- **Outcome**：开放式网络靠规模与互通成为默认；闭环小网络规模受限。  
- **T0 环境**：跨地区/跨商户支付需求上升。  
- **Killer Assumption**：闭环网络靠品牌即可维持规模。  
- **Wedge**：商户覆盖不足、可用性差。  
- **Flywheel**：商户多 → 用户多 → 交易多 → 商户更多。  
- **First Tripwire**：商户覆盖率下降、交易失败率上升。  
- **Catastrophic Bound**：关键商户或地区无法覆盖导致用户流失。  
- **Death Spiral**：覆盖少 → 使用少 → 交易少 → 商户退出 → 覆盖更少。  
- **Noise**：短期补贴拉新。  
- **最短探针（30d）**：测“商户覆盖率 + 日活交易率”，低于阈值即判网络效应不足。  

### 28.2 互通标准（相对成功） vs 钱包孤岛（失败模式）
- **Outcome**：互通标准提升普及率；孤岛钱包在互通不足下增长受限。  
- **T0 环境**：多钱包并存、用户负担高。  
- **Killer Assumption**：靠单一生态可长期封闭运行。  
- **Wedge**：跨钱包交易不可用导致流失。  
- **Flywheel**：互通↑ → 可用场景↑ → 用户↑ → 互通更强。  
- **First Tripwire**：跨钱包失败率高、用户多钱包切换成本高。  
- **Catastrophic Bound**：主要场景转向互通网络，孤岛边缘化。  
- **Death Spiral**：场景少 → 使用少 → 商户少 → 场景更少。  
- **Noise**：短期用户数增长。  
- **最短探针（7d）**：统计“跨钱包成功率 + 互通商户占比”，低于阈值即判孤岛风险。  

### 28.3 一体化收单工具（相对成功） vs 传统割裂收单（失速）
- **Outcome**：一体化收单与风控提升商户转化；割裂流程导致成本高与失败率高。  
- **T0 环境**：在线支付与订阅增长，欺诈风险上升。  
- **Killer Assumption**：商户愿意长期忍受高失败率与多供应商。  
- **Wedge**：失败率与欺诈损失侵蚀利润。  
- **Flywheel**：成功率↑ → 商户留存↑ → 交易量↑ → 风控与成功率更好。  
- **First Tripwire**：支付成功率下降、欺诈率上升。  
- **Catastrophic Bound**：合规成本与失败率导致商户大规模迁移。  
- **Death Spiral**：失败率↑ → 商户流失 → 交易量↓ → 风控资源↓ → 失败率更高。  
- **Noise**：单月交易量冲高。  
- **最短探针（48h）**：监测“支付成功率 + 拒付率”，超过阈值即判工具链失效。
