## 23.x 国防/航天/政府采购（06 硬化案例）

### 23.1 SpaceX（成功） vs 传统一次性发射（失速）
- **Outcome**：可复用与垂直整合大幅降低发射成本；一次性发射在成本曲线中失速。  
- **T0 环境**：商业航天需求增长，政府预算有限。  
- **Killer Assumption**：传统成本结构可以长期维持市场份额。  
- **Wedge**：可复用带来成本断崖下降。  
- **Flywheel**：成本↓ → 订单↑ → 规模↑ → 成本更↓。  
- **First Tripwire**：单位发射成本持续高于市场均值。  
- **Catastrophic Bound**：无法在招标中竞争或失去关键客户。  
- **Death Spiral**：订单↓ → 规模↓ → 成本更高 → 订单更少。  
- **Noise**：单次发射成功率。  
- **最短探针（30d）**：测“每公斤入轨成本 + 订单赢单率”，低于阈值即判失速。  

### 23.2 高复杂项目（失败模式） vs 可迭代平台（相对成功）
- **Outcome**：高度定制的大项目易超支延期；可迭代平台能持续交付。  
- **T0 环境**：需求不断变化、合规与安全要求上升。  
- **Killer Assumption**：一次性大项目可以完全定义需求并按期交付。  
- **Wedge**：需求变化导致反复返工。  
- **Flywheel**：迭代交付 → 反馈闭环 → 成本可控 → 再迭代。  
- **First Tripwire**：需求频繁变更、里程碑连续延期。  
- **Catastrophic Bound**：总成本超过预算阈值且无法缩减范围。  
- **Death Spiral**：延期 → 成本↑ → 审批更严 → 延期更长。  
- **Noise**：短期“技术突破”宣传。  
- **最短探针（7d）**：统计“变更单数量 + 延期频率”，超过阈值即判项目病。  

### 23.3 快速采购（相对成功） vs 传统流程（失速）
- **Outcome**：快速采购能抓住窗口；传统流程在需求变化中失去机会。  
- **T0 环境**：威胁环境快速变化，技术周期缩短。  
- **Killer Assumption**：传统采购流程可以跟上技术迭代。  
- **Wedge**：时间延迟导致需求过时。  
- **Flywheel**：快速迭代 → 实战反馈 → 改进 → 更快迭代。  
- **First Tripwire**：采购周期超过技术迭代周期。  
- **Catastrophic Bound**：交付时技术已过时且无法部署。  
- **Death Spiral**：延期 → 需求变化 → 返工 → 再延期。  
- **Noise**：阶段性通过审计。  
- **最短探针（48h）**：测“采购周期/技术周期比”，超过阈值即判失效。
