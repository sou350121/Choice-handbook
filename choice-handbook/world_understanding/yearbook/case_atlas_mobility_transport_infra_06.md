## 15.x 出行/交通/城市基础设施（06 硬化案例）

### 15.1 Tesla（成功） vs Better Place（失败）
- **Outcome**：Tesla 以垂直整合+充电网络建立壁垒；Better Place 因换电基础设施与商业模型失配失败。  
- **T0 环境**：电池成本快速下降但充电网络稀疏。  
- **Killer Assumption**：高 CAPEX 的换电网络可以先规模化再找需求。  
- **Wedge**：电池与软件垂直整合 + 充电网络形成闭环。  
- **Flywheel**：车队规模↑ → 充电利用率↑ → 网络回本↑ → 车队规模↑。  
- **First Tripwire**：网络利用率低、资本开支超预算、政策与土地审批延迟。  
- **Catastrophic Bound**：基础设施成本长期高于车辆毛利与现金流。  
- **Death Spiral**：资金紧 → 扩张慢 → 需求弱 → 利用率更低 → 资金更紧。  
- **Noise**：短期交付或单季利润波动。  
- **最短探针（30d）**：测“充电/换电网络利用率 + 单站回本周期”，超过阈值即判不可扩张。  

### 15.2 Uber（成功） vs 传统出租车牌照体系（失速）
- **Outcome**：Uber 以调度与动态定价扩张；传统牌照体系在供给弹性上失速。  
- **T0 环境**：智能手机普及，城市出行需求上升。  
- **Killer Assumption**：监管不会对平台化调度进行严格限制。  
- **Wedge**：动态定价 + 司机供给激励。  
- **Flywheel**：乘客↑ → 司机↑ → ETA↓ → 乘客↑。  
- **First Tripwire**：司机收入下降、司机流失率升高、监管诉讼密集。  
- **Catastrophic Bound**：核心城市被禁或被严格配额，供需无法平衡。  
- **Death Spiral**：补贴↓ → 司机↓ → ETA↑ → 需求↓ → 补贴更高 → 亏损更大。  
- **Noise**：补贴驱动的 GMV 冲高。  
- **最短探针（7d）**：监测司机留存、补贴/单量、平均等待时间，任一失控即判调度飞轮断裂。  

### 15.3 Shinkansen（成功） vs Concorde（失败）
- **Outcome**：新干线以容量与可靠性成为公共基础设施；协和因成本与需求结构失败。  
- **T0 环境**：中短途需求大、能源成本上升。  
- **Killer Assumption**：速度溢价足以覆盖高成本与低容量。  
- **Wedge**：高频可靠服务覆盖大众需求。  
- **Flywheel**：可靠性↑ → 乘客↑ → 收入↑ → 维护与扩张↑ → 可靠性↑。  
- **First Tripwire**：载客率低于盈亏线、维护成本持续上升。  
- **Catastrophic Bound**：安全事故或监管禁飞/停运。  
- **Death Spiral**：载客低 → 提价 → 载客更低 → 成本更高 → 关停。  
- **Noise**：媒体热度与技术炫技。  
- **最短探针（30d）**：测“载客率 vs 维护成本”是否过盈亏平衡线。  

### 15.4 Citi Bike（相对成功） vs Ofo（失败）
- **Outcome**：有站点与城市协作的共享单车更可持续；无序投放在运维与资产折旧中崩溃。  
- **T0 环境**：城市微出行需求旺盛，资产密集。  
- **Killer Assumption**：规模可以自动降低运维成本。  
- **Wedge**：站点治理与城市合作降低损坏率。  
- **Flywheel**：可用性↑ → 骑行↑ → 收入↑ → 维护↑ → 可用性↑。  
- **First Tripwire**：损坏率上升、维护积压、城市执法升级。  
- **Catastrophic Bound**：现金流无法覆盖维修与回收成本。  
- **Death Spiral**：坏车↑ → 体验差 → 骑行↓ → 收入↓ → 维护更少 → 坏车更多。  
- **Noise**：短期投放数量。  
- **最短探针（7d）**：监测“完好车占比 + 维修响应时长”，低于阈值即判不可持续。
