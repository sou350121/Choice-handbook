# 06 时空穿越者式极限思索（T0/Now）× 盡頭推演引擎（Endgame Simulator）

> 目标：用**两次视角切换（T0→Now）** + **闸门（Gate 1/2/3）** + **Endgame Map（先 Catastrophic）**，在不靠事后诸葛的前提下，识别“BTC 级（10x~1000x 非线性）”机会，并把输出变成**决策工程**：假设 → 探针 → 分段下注 → 退出/重启。
>
> 适用：你给一篇文章/线索，直觉像“新 Bitcoin”但主流尚未理解；你要的是**可执行策略**与**可证伪更新**。  
> 不适用：纯短线交易；或你无法/不愿做任何取证与复盘（那只能讲故事）。

---

## 核心原则（先锁灾难上界，再谈收益）

- **先 Catastrophic bound**：先写“不可逆红线”，再写上行空间。  
- **每个关键断言三件套**：证据（可追溯）/ 反证条件（可观测）/ 下一步取证（探针）。  
- **禁止事后诸葛**：T0 只用当时合理可得信息；Now 才允许利用“今天走到哪”。  
- **时变信息强制核验**：价格/市值/链上/用户/融资/政策/版本/人物职位等 → 必须 web search 并标注日期；否则不得 Gate 3。  

### 三视角切换（借鉴：穿越者 / 机制 / 上帝）

这不是文风，而是“强制换镜头”的防偏差机制：

- **穿越者视角（T0）**：只用当时可得信息与约束（对应第 1 节）。  
- **机制视角**：用经济学/博弈论/社会动力学拆解激励、参与者与传播（主要落在第 2/2.5/3/5/8 节）。  
- **上帝视角**：看穿盲区，给出最短通关路径与最短反证路径（主要落在第 6/8 节）。  

---

## 三闸门（只能三选一）

- **Gate 1 淘汰**：反证路径太短 / 红线风险过高 / 无可验信号（Time-to-Signal 太长且不可控）。  
- **Gate 2 买信息期权**：小成本换关键证据（探针+时间盒+退出阈值），先把 20% 推到 50%。  
- **Gate 3 分段下注**：已具备清晰升维信号（20→50→80 的升级条件写死），且 Catastrophic bound 可控。

## 固定输出格式（每次必交付，不可省略标题）

> 下面是“骨架模板”。你每次输出都必须保留这些标题；内容允许短，但不能把标题省掉。

### 0) 快速结论（1段）
- Bitcoin 级潜力：A / B / C / D（见下方“分档说明”）
- 倍率判断（机制上限，不承诺价格路径）：10x / 100x / 1000x
- 当下决策：Gate 1 / Gate 2 / Gate 3
- 一句理由（只写最核心那条）

**分档说明（必须按此口径写，避免含糊）**
- **A（更像 1000x）**：具备成为跨行业基础设施/结算层/协议标准级“默认层”的路径与信念/合法性飞轮；且 catastrophic bound 可控
- **B（更像 100x）**：更可能成为一个大行业默认 SoR/中枢（或多行业关键节点），存在可验证的网络效应/数据壁垒/定价权
- **C（更像 10x）**：强产品/强工具/局部垂直 SoR，但难跨行业标准化，或价值捕获不稳定
- **D（更像噪声）**：缺硬约束/飞轮/可验证信号，更多是叙事与情绪

### 1) T0 时空重建（当时环境）
- 材料定性（先贴标签，避免拿错工具）：类型/阶段/可信度/时效（可复用 `frameworks/05_reasoning_toolkit.md` 的 A1–A3）
- **T0-Now（此刻当时）**：技术 / 基建 / 监管 / 资金 / 社会心理（每项 1–3 点；只用当下可得信息）
- **T0-Analog（类比回放，必输）**：
  - 选 2–3 个 **结构相似类比物**（Primitive/Wedge/Flywheel/Moat 风险相似）
  - 对每个类比物，回放其“早期”：
    - 早期领先指标（哪些信号是真的）
    - 早期噪声/宣传（哪些信号会误导）
    - 最常见失败模式（死法）
  - **Roadmap 映射（必输）**：把类比物的演化阶段映射到本分析的 `Phase0→1→2→3→4`（见第 10 节），写清楚：
    - 现在处在 Phase 几？
    - 下一阶段最小里程碑是什么？
    - 若卡死，最可能卡在哪个摩擦？
- **推荐输出格式（强烈建议）**：用一张“类比→路线图映射表”把信息压缩成可执行门禁：

| 类比物 | 对应 Phase（Phase0→4） | 早期领先指标（2–3） | 早期噪声（1–2） | 常见死法（1–2） | 映射到本项目：下一阶段最小里程碑 | 最可能卡点 |
|---|---|---|---|---|---|---|
| A | Phase? | ... | ... | ... | ... | ... |
| B | Phase? | ... | ... | ... | ... | ... |
| C（可选） | Phase? | ... | ... | ... | ... | ... |
- **ASCII 结构快照（必输）**：至少 1 张 ASCII 图，把“核心/边界/压力/矢量方向”画出来（用于读者快速理解结构形状）。
- 当时大多数人忽略/误解什么？
- 当时最大的摩擦/阻力在哪？

### 2) 第一性原理：它解決什麼「不可妥協問題」？
- Problem：谁痛？痛到什么程度？为什么现有方案不可行？
- Primitive（新原语）：它创造了什么新能力？（一句话）
- Necessities（必要条件×3）：缺任何一个就不可能成功
- Flywheel（技术/经济飞轮）：用户→价值→安全/可用性→用户（用箭头链条写）

### 2.5) 社会底座与合法性（Belief Flywheel，必输）
- S1 合法性裂缝（Legitimacy Vacuum）：承接哪条制度裂缝？扩大还是缩小？
- S2 最早部落（Tribe）：谁会先信？身份动机？愿意付出什么代价？
- S3 神话与仪式（Myth/Ritual）：一句话神话 + 可重复节律/符号
- S4 敌对叙事（Narrative Enemy）：最强反叙事是什么？如何把攻击反脆弱化？
- S5 协调红利（Coordination Gain）：降低哪种协调成本？让陌生人一起做到什么？
- S6 信念钩子（Belief Hook，仅 1 条）：为什么能长期持有/建设/忍受波动？
  - 若缺 S6：**不得 Gate 3**，只能 Gate 2
- Belief Flywheel：裂缝/不信任→象征/身份→传播→社会证明/资本→更强信念（箭头链条）
- 耦合点：技术/经济飞轮与信念飞轮在哪里耦合？（例：价格/注意力→安全/算力）

### 3) 竞争与护城河（Moat）
- 网路效应类型：直接/间接/资料/协议/生态（选并解释）
- Wedge（最早必用场景，仅 1 条）：谁在什么场景下非用不可？
- 可替代性：换团队/换协议能否同样做？差异在哪？
- 失败模式：最可能死于什么（技术/治理/监管/分发/信任/激励）
  - 机制补充（可选但推荐写 1–2 条）：参与者与激励矩阵 / 攻击向量与成本收益 / 谢林点为何是它而不是竞品

### 4) 盡頭推演（Endgame Map：先 Catastrophic）
> 目标：把“尽头”写成可执行门禁，而不是文学感受。

#### 最可能的尽头（必输：短/中/长）

- **短期（0–12 个月）**：最可能发生什么？最早的停机点是什么？  
  - **6 个月社会需求雷达（强制，写“一线/二线/深层”）**：  
    - **一线需求（直接、立刻、刚性）**：政策/医疗/供应/出行/安全/信息等最先被“硬约束化”的需求是什么？（列 3–5 条）  
    - **二线需求（派生、加速、补洞）**：一线需求带来的“次级市场/工具/服务”需求是什么？（列 3–5 条）  
    - **深层高概率需求（结构性，不靠单点事件）**：即使事件退潮也会留下来的需求是什么？（列 1–3 条）  
    - **T0-Analog 映射（必输）**：选 2–3 个“结构相似年份/事件”做对照（例如 2003 SARS）：  
      - 当时的一线需求是什么？二线需求是什么？  
      - 哪些看似重要但只是噪声？  
      - 把“当年需求→今天需求”写成 3 条可验证预测（含最早信号）。  
- **中期（1–3 年）**：最可能收敛到什么稳态？最可能的卡点是什么？  
- **长期（3–10 年）**：用四象限写终局：  
  - **Catastrophic**：不可逆出局（安全/共识/法律合规/声誉/治理崩）  
  - **Bear**：被替代/需求不足/信任崩  
  - **Base**：小众但稳定的有用工具  
  - **Bull**：新基础设施/新货币/新垄断平台  

#### 停止机制（必输）

- 写 3 条“停止机制/停机点”（每条必须可观测）：出现就**退出或降级 Gate**（默认先降级到 Gate 2/1）。

#### 「本来可以发生却没发生」（必输）

- 写 1 条：看似有窗口却没发生的“更大叙事”，并解释**核心阻力/硬约束**是什么。

#### 「如果成功」的终极状态（必输）

- 用 3–5 句写清：成功不是“价格”，而是**世界默认被改写成什么**（谁得利、谁失利、什么规则改变）。

### 5) 非对称性评分（0–5）
逐项打分 + 一句理由：
- 尾部上行（Tail Upside）
- 生存力（Survivability）
- 早期可验证性（Time-to-Signal）
- 扩散/分发（Distribution）
- 监管/对手风险（Regulatory/Adversarial）
- 反身性（Reflexivity）
- 叙事共鸣（Narrative Resonance）
- 合法性可持续（Legitimacy Durability）
输出总评：A/B/C/D

（必写）七层共振检查（Yes/No + 一句理由，越短越好）：
- 数学/物理层（机制是否稳固）
- 技术层（可行且难复制）
- 博弈论层（激励自洽/谢林点）
- 经济学层（价值创造/供需/网络效应）
- 社会动力学层（传播/部落/反脆弱）
- 意识形态层（信念承载）
- 时代精神层（社会情绪/合法性裂缝）

共振结论：对齐层数 → 机会级别（仅作筛选，不承诺收益）：
- 7 层：更可能“改变世界级”
- 5–6 层：更可能“改变人生级”
- 3–4 层：更可能“不错的投资/产品”
- 1–2 层：更可能“有趣但不非线性”

### 6) 决策工程：探针 + 分段下注（反证优先）
- 先列 3 条最短反证路径（如何最快证明它错）
- 再列 3 条支持路径
- 只保留 1 条最短路探针（48h/7d/30d 三选一）：行动/可观测信号/退出阈值（写死）
- 分段策略（仅 Gate 3 可写）：
  - Buy Option：投入上限、时间盒、验证信号
  - Add：20%→50%→80% 升级条件（写死）
  - Exit：立刻撤退信号
  - Restart：只有哪些新证据出现才允许重启

### 7) 证据表（Evidence Table：5–10 条）
- 主张 | 文内短引句/可观测事实 | 判断（支持/反对/不确定）| 可验证方式（下一步取证）

### 8) 上帝视角洞察（必输）
#### 8.1 盲点揭示（皮肉）
- 3 个常见误区：必须落到“可观察的行为/叙事模式” + 解药（信号/探针/闸门）
#### 8.2 通关秘籍（最高效掌握路径）
- 最少步骤掌握骨架：每步必须交付产物（表/指标/访谈问题/仪表板/小作品）
#### 8.3 一句话本质
- “用【Primitive + (Belief Hook) + Flywheel + Moat】去求解【根本问题】。”

### 9) 对比与估值（必输：100x vs 1000x + 天花板）
#### 9.1 类比对照（Comparables，至少 2–3 个）
- 相同骨架（Primitive/Wedge/Flywheel/Moat/主要摩擦）+ 关键差异 + 类比最可能误导点
#### 9.2 市值估计（Bull/Base/Bear/Cat）
- 先给当前基数（token：流通市值/FDV/供给曲线；股权：估值/收入/增速）——必须 web search 核验并标注日期
- 每个情景至少两种方法交叉验证：TAM/SAM/SOM；价值捕获；网路效应溢价
- 输出数字区间（低/中/高）+ 关键假设 + 反证条件
#### 9.3 100x vs 1000x 判断（Growth Multiple）
- 计算：100x/1000x 目标市值（基于当前市值）
- 判断哪一档“物理上可行”：需要成为全球底座/行业标准/小众工具？
- 若数据不足：降级 Gate 2，只列必补资料清单

**关键补充（强制写清，避免“价格=百倍”误读）**
- **机制百倍/千倍**：指它在终局里能否成为“默认层”（基础设施/标准/结算/SoR 中枢）并拥有持续价值捕获；不是指你一定能拿到 100x/1000x 的价格路径
- **交易百倍/千倍**：取决于你能否在正确时间窗持有（并穿越波动/监管/叙事反转），以及是否存在被稀释/被替代/被合规击穿的路径

**1000x（A 档）必要条件（任一缺失就不得写 1000x）**
1) **默认层地位**：跨行业/跨地域可扩散，且存在清晰标准化路径（协议/结算/接口/生态）
2) **强反身性飞轮**：注意力/资本/安全/生态之间形成自我增强闭环（且能解释“为什么会越走越强”）
3) **合法性/信念飞轮**：能承接制度裂缝或信任裂缝，并长期维持（不是一次性营销）
4) **catastrophic bound 可控**：最致命的监管/安全/治理红线有明确缓释路径与可观测信号

### 10) 发展路线图像（必须像图）
- 输出 ASCII/文本路线图：Phase0→1→2→3→4（每阶段：里程碑/依赖/可观测信号/失效信号/Gate 决策）
- 两条并行轨道：
  - A) 技术/产品/经济轨（PMF/分发/价值捕获/安全）
  - B) 合法性/社会轨（部落→攻击存活→符号仪式→跨部落→制度化叙事）

- 最终一句话（可贴决策日志）：【候选】—【A/B/C/D】—【最短探针】—【停机点】—【下次回看日期】

## 附录 A：a16z “三条路”→ Gate 打分卡（SoR / 劳动力替代 / 围墙花园数据）

> 用途：把 a16z / Rampell 的框架从“观点”升级为**可执行筛选器**。  
> 使用场景：当你在看一家公司/产品/赛道是否具备长期护城河时，用这张卡直接给 Gate 1/2/3。

### A1) 三条路（先判“它在走哪条”）

在下面三类里，至少命中一类（可多选）：

1) **AI 原生改造（retrofit）**：把既有软件改写为 AI-native，并推动 seat→usage/outcome 的定价迁移。  
2) **替代劳动力（replace labor）**：软件吃掉岗位 70–90% 的核心职责，价格锚定在人力与结果之间。  
3) **围墙花园数据（walled garden）**：独家授权/采集网络/闭环结果数据，让“模型同质化”也撬不动壁垒。  

若三类都不命中：通常是“薄工具/薄包装”→ **倾向 Gate 1**（除非你能证明它的分发或社区本身是护城河）。

### A2) SoR（System of Record）检查（这是护城河的“栖身地”）

回答 3 个问题（每题写：证据 / 反证 / 探针）：

1) **它是否进入交付与审计关键路径？**（用户离不开它来“完成事”而非“看信息”）  
2) **迁移成本是否真实上升？**（数据迁移、集成、培训、流程再造、合规审计）  
3) **它是否在产出“结果记录”而不只是“建议”？**（结果=能被复盘的完成/失败与原因）

若 1) 与 3) 都为否：基本不可能是 SoR → **至少 Gate 2，常见 Gate 1**。

### A3) 数据壁垒检查（最值钱的是“别人拿不到的数据”）

把数据按来源分三类（至少命中一类才算“壁垒成立”）：

- **闭环结果数据**：系统在工作流里产生“结果”（修复/成交/履约/退款/复发等），外部大模型拿不到。  
- **独家授权数据**：合同与法务上排他（或足够长期/足够难替代）。  
- **采集网络+清洗对齐**：长期布点/采集/清洗/对齐/可访问化（公共资源炼成资产）。  

若数据来源只是“用户输入文本/公开网页”且无闭环：通常不足以构成壁垒 → **倾向 Gate 2**。

### A4) 定价迁移（seat → usage/outcome）检查

写清楚它现在的定价与未来的目标定价：

- **当前定价**：seat / usage / outcome / 混合  
- **价值对齐**：客户愿意为结果付费吗？（而不是为闲置席位付费）  
- **可验证证据**（必须有其一）：公开条款/财报口径/客户合同案例/明确的续费与扩张数据

若你拿不到可验证证据：不得 Gate 3（只能 Gate 2）。

### A5) 三闸门输出规则（把打分卡转成行动）

- **Gate 1（淘汰）**：  
  - 不命中三条路；或没有 SoR 迹象；或数据壁垒不成立且分发也不构成护城河。  
  - 最短反证路：同类功能被平台/竞品复制后，留存与定价权迅速塌陷。

- **Gate 2（买信息期权）**（默认）：  
  - 命中至少一条路，但关键证据不足（SoR 证据/数据壁垒/定价迁移缺一）。  
  - 48h/7d/30d 最短探针：补齐“3 个证据”：SoR 证据、数据壁垒证据、定价迁移证据（任一补不齐就降级）。

- **Gate 3（分段下注）**（门槛很高）：  
  - SoR 明确成立 + 数据壁垒至少一种成立 + 定价迁移有可验证证据 + catastrophic bound 可控。  
  - 必须写 20%→50%→80% 的升级条件（写死）与退出/重启阈值。

### A6) 10x / 100x / 1000x 的粗分档（机制上限，不承诺收益）

用“它能占据的利润池/标准地位”来分档（先别谈价格路径）：

- **更像 10x**：在一个垂直场景成为强工具或局部 SoR，但难跨行业标准化。  
- **更像 100x**：成为一个大行业的默认 SoR（或多行业的关键中枢），并拥有难替代数据壁垒与定价权。  
- **更像 1000x**：需要成为跨行业的基础设施/结算层/协议标准级“默认层”（极少见；且通常需要强合法性/信念层支持）。  

> 现实约束：对大多数 AI 应用与企业软件而言，除非出现强“信念/合法性飞轮”与全球标准化路径，否则通常不应轻易打到 1000x 档。
