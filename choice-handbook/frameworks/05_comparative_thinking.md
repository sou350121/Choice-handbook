# 05 比较思考：用发展史与对比技术推演当下与未来

当你讨论一个新事物（技术、产业、策略、商业模式），最大的风险是：
- 只看当下噪声，忽略长期结构
- 只看远方叙事，忽略现实约束
- 把类比当结论，导致方向性幻觉

本篇提供一个“思考更自由，但输出更可靠”的方法：**发展史 + 结构类比 + 情景推演 + 观察信号**。

---

## 什么时候该用？

尤其适用于：
- 技术发展路线：选哪条栈、押哪种范式（开源/闭源、端侧/云侧）
- 产业趋势：泡沫、挤压、平台化、监管变化
- 研究方向：哪些会成为“基础设施”，哪些只是阶段性热点

---

## 核心原则：类比是“生成假设”，不是“输出答案”

你可以大胆类比、发散推演，但最后必须回到：
- 当下证据（可观测）
- 现实尽头（停止之处）
- 可逆实验（买期权）

---

## 方法：H→M→F（History / Mapping / Future）

### 1) History（发展史）
把“对比对象”拆成 3–5 个阶段（按机制划分，不按年份）：
- 萌芽：技术可行但成本高、体验差、用例窄
- 扩散：工具链成熟、人才涌入、用例爆发
- 挤压：同质化与价格战，赢家吃掉分发/数据/算力
- 平台化：从“产品”变成“基础设施”
- 常态化：监管、标准、成本结构稳定，创新转向边缘与垂直

> 提示：不是每个对象都有全部阶段，但你要写清楚“它卡在哪个阶段，为什么卡住”。

### 2) Mapping（结构映射）
把旧对象的关键变量映射到当下主题（建议 5–8 个）：
- 供给：核心能力/算力/数据/算法
- 分发：入口、渠道、平台、默认选项
- 成本：单位成本、边际成本、规模经济
- 生态：工具链、人才、标准、社区
- 监管与风险：安全、隐私、合规、外部性
- 用户心智：信任、习惯、切换成本
- 商业模式：付费方式、议价权、复利来源
- 竞争格局：赢家通吃 vs 多极共存

### 3) Future（情景推演）
给 2–3 个情景（保守/基准/激进），每个情景写：
- 一句话描述
- 核心机制（为什么会发生）
- 3 个观察信号（信号出现代表该情景更可能）

### 4) Anti-analogy（反类比）
写清楚“哪里不像”，并说明它会怎样改变推演。

---

## 示例：LLM 的当下与未来，类比几年前的推荐算法（结构层面）

下面不是结论，只是演示写法。

### 发展史（推荐算法的结构阶段）
- 早期：特征工程 + 小规模数据，效果提升明显但体验不稳定
- 扩散：数据规模与在线系统成熟，AB 测试与反馈闭环成为核心资产
- 挤压：同质化后，赢家靠分发入口 + 数据网络效应
- 平台化：推荐成为基础设施（广告、内容、电商、社交的“默认引擎”）

### 映射到 LLM
- 供给：模型能力/算力/数据（尤其是高质量数据与后训练能力）
- 分发：默认入口（搜索、操作系统、办公套件、IDE、App 内置）
- 成本：推理成本下降速度决定“无处不在”还是“谨慎调用”
- 生态：工具链（Agent、RAG、评测、对齐、安全）与人才供给
- 监管：版权、隐私、内容安全、关键行业合规

### 未来情景（示例）
- 保守：LLM 主要停留在“增强工具”，增量价值稳定但不颠覆主入口  
  - 信号：推理成本下降变慢；高风险行业合规成本上升；用户留存趋稳
- 基准：LLM 成为多个系统的“默认层”，但被平台化入口吸收  
  - 信号：OS/办公/搜索深度内置；企业标准化评测与采购出现；中间层工具链成熟
- 激进：LLM + Agent 形成新交互范式，部分旧入口被替代  
  - 信号：端侧能力跃迁；可靠性/可控性显著提升；新平台出现并获得分发优势

### 反类比（哪里不像）
- 推荐算法的核心资产是“用户行为数据 + 分发入口”；LLM 的核心还包括“通用能力 + 后训练 + 安全边界”。  
这会让 LLM 的竞争不只发生在分发，还发生在“能力与对齐”的技术栈上。

---

## MUST / SHOULD / MAY（针对比较思考）

### MUST
- 明确标注：哪些是事实，哪些是类比推演（假设）。
- 写出反类比（哪里不像），避免“类比即结论”。

### SHOULD
- 优先选“结构相似”的对比对象（机制、约束、分发与成本结构）。
- 用情景 + 信号来写未来，而不是单一预测。

### MAY
- 画一张 2×2：成本下降速度 × 分发入口强度，快速枚举情景。
- 给每个信号设置阈值（例如：推理成本/延迟达到某水平才成立）。

---

## 下一步实验

选一个你关心的主题（例如 LLM、机器人、量化策略、内容平台），用 30 分钟写：
1. 对比对象是谁？（2 个候选）
2. 发展史阶段是什么？（3–5 段）
3. 你认为当下处于哪个阶段？依据是什么？
4. 未来 3 个情景 + 每个 3 个信号
5. 写清楚反类比（哪里不像）

写完把它放进你的 `worksheets/decision_one_pager.md` 的“关键假设/未来信号”里，作为“买期权”的依据。
