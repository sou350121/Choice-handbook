## 1) 这篇材料在说什么（抽象成骨架）

- **定位**：类似 Claude Cowork 的“桌面级多智能体工作流”，强调“AI 真正执行任务而不只是聊天”。
- **功能叙事（材料声称，部分可核验）**：
  - 多角色 worker（开发/搜索/文档/多模态）并行协作
  - 本地优先：支持本地模型（如 Ollama），数据不出本地
  - MCP：可接入大量工具（浏览器、代码执行、第三方 SaaS、内部 API）
  - 可云可本地：既提供 cloud，也提供 self-hosted 社区版

## 2) 可抽取资产（Definitions / Variables / Mechanisms）

### 2.1 定义

- **Cowork（桌面协作代理）**：在用户授权的文件/工具范围内，能规划并执行多步任务的“数字同事”。
- **Multi-agent workforce**：不是多开聊天窗口，而是“任务拆解→并行执行→结果汇总→失败重试/升级”的调度系统。
- **Local-first**：默认把数据边界放在本地（文件、截图、日志），云只作为可选加速/托管。
- **MCP 工具协议**：把“能做什么”外包给工具生态（网页、文件、Notion/Slack/Google 等），模型本体更易替换。

### 2.2 关键变量（后续验证/反证会用到）

- **可控性**：权限边界、确认机制、审计日志、回滚与安全护栏
- **可扩展性**：工具接入成本（MCP server 数量、质量、可维护性）
- **可靠性**：并行后是否可复盘（失败分类、重试策略、幂等性）
- **隐私与合规**：本地模式是否真正不出网；企业版 SSO/访问控制
- **许可风险**：所谓“开源”是否对商业/人数/托管有限制（许可证决定能否成为事实标准）

### 2.3 机制（为什么会火，也为什么会死）

- **火的机制**：把“执行权”产品化（文件/终端/浏览器/工具），并用 MCP 降低工具集成摩擦。
- **死的机制**：
  - 平台吞并：Claude Desktop / OS 级 agent 把执行权做成系统能力，独立 app 被挤压
  - 安全事故：提示注入/越权/误删误改导致信任坍塌
  - 许可证与生态：非 OSI 许可导致社区贡献受限，难形成事实标准

## 3) 关键主张（Claims）与可验证预测（Predictions）

1) **Agent 的主战场从“回答”转向“执行”**  
   - 预测：桌面/文件/工具链的“授权+审计+回滚”会成为标配。
2) **MCP（或类似协议）会成为工具生态的事实接口**  
   - 预测：工具生态会快速扩张，但也会出现质量分层与安全基线需求。
3) **Local-first 会成为企业与个人的关键分岔**  
   - 预测：对隐私敏感场景更倾向本地/自托管；但性能与维护成本会反向约束 adoption。
4) **最大风险：价值捕获与信任崩**  
   - 预测：若出现安全事故或越权行为，用户会快速把它降级为“只读助手”。

---

## 4) 不确定性与盲区

- **“开源”程度与许可证限制**：这决定它能否成为公共基础设施，而不只是营销。
- **桌面执行权的安全模型**：权限、确认、审计、回滚是否可用（这是 catastrophic bound）。
- **并行多智能体的真实收益**：是否只是“并发噪声”，还是能稳定减少时间与错误率。
