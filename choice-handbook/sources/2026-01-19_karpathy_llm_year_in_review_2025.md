## 文章主张（作者视角）

作者列出 2025 年 6 个“范式转变”（他认为重要且令人惊讶）：

1) **RLVR（可验证奖励强化学习）成为新阶段**：在可自动验证的环境里长时间优化，促使模型发展“推理策略”，并引入“测试时计算量/思考时间”的新旋钮。  
2) **“幽灵 vs 动物”与不均衡智能**：LLM 不是培育动物，而像“召唤幽灵”；智能形态不均衡，在可验证领域能力飙升，但也容易在安全/越狱等处失足。  
3) **Cursor 式应用层**：出现更“厚”的应用层（上下文工程 + 多次调用编排 + GUI + 自主性滑块），人们开始谈“用于 X 的 Cursor”。  
4) **Claude Code 与“住在你电脑里的 AI”**：关键不在云/本地的地理位置，而在本地已有的安装、上下文、数据、密钥、配置与低延迟交互。  
5) **氛围编码**：仅用自然语言就能构建程序，“代码变得更便宜、临时、可塑、一次性”。  
6) **LLM 的 GUI（nano banana）**：文本像早期计算的控制台；人更偏好视觉/空间信息；多模态与世界知识纠缠在权重中，暗示新的交互范式。

## 可抽取资产（Asset Extraction）

### 定义（概念边界）

- **RLVR**：基于可自动验证奖励的强化学习（相对 RLHF 更“客观/不可操纵”的奖励假设）。
- **测试时计算/思考时间**：能力作为测试时计算量的函数可控（作者称为新旋钮/扩展定律相关）。
- **不均衡智能**：在可验证领域出现“尖刺式”能力提升，整体能力分布不平滑。
- **应用层（Cursor-like）**：为特定垂直整合编排 LLM 调用（上下文工程、DAG 编排、GUI、自主性滑块）。
- **本地 Agent**：运行在开发者机器上，直接使用私有环境/数据/配置/密钥并提供低延迟交互。
- **氛围编码**：通过自然语言快速生成/改写代码，代码更像一次性媒介。
- **LLM GUI**：面向人类偏好的信息表达形式（图像/白板/动画/应用）而非纯文本。

### 变量（关键量）

- 训练侧：预训练算力 vs RLVR 算力分配、RLVR 时间长度、可验证环境覆盖面
- 推理侧：测试时计算预算、长推理轨迹长度、纠错策略密度
- 评测侧：可验证基准占比、刷榜压力、合成数据与基准邻域过拟合
- 产品侧：上下文工程质量、工具编排复杂度、成本/延迟权衡、自主性滑块位置
- 交互侧：文本 vs 视觉/空间 UI、用户阅读与输入成本
- 安全侧：越狱攻击成功率、数据泄露风险

### 机制（因果链 / 互动算子）

- **可验证奖励 + 长期优化 → 推理策略涌现**（分解步骤、反复推敲、纠错）
- **可验证基准构成评测空间 → RLVR 容易在邻域“长尖刺”** → 基准信任下降
- **本地环境上下文丰富 + 低延迟交互 → Agent 更易落地**（即便云端更像“终局”）
- **自然语言生成能力跨过阈值 → 代码生产边际成本下降** → 更多临时/一次性软件被生产
- **多模态联合能力 → 交互形态从文本走向 GUI**（图像、白板、应用等）

### 反例/失败模式（尽头线索）

- 基准/评测被系统性“刷榜”，导致优化目标偏移、真实能力难评估
- 能力分布不均衡：局部很强但整体不稳，安全/可靠性成为现实停止处
- Agent 依赖私有上下文与权限，带来数据泄露与不可控执行风险

### 可验证预测（假设/推演）

- 未来一段时间能力提升更多来自“训练与推理阶段的长优化/长思考”而非单纯扩大参数规模。
- “LLM 应用层”会继续变厚：通过私有数据/执行器/反馈回路把通用模型组织成垂直从业者。
- 交互会从“聊天”为主逐渐走向多模态 GUI（白板/幻灯片/应用生成与编排）。
