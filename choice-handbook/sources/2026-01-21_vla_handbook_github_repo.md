## 1) 这份材料在说什么（只留结构性主张）

- 这是一个面向 **VLA（Vision-Language-Action）/具身机器人** 的全中文、工程导向知识库，覆盖：
  - 理论（theory）
  - 真机部署（deployment）
  - 速查表（cheat-sheet）
  - 题库/实战（question-bank）
  - 系统设计（system-design）
  - 公司与求职（companies）
- 它的“暗含立场”是：VLA 不再只是论文阅读，而是**硬件-数据-模型-部署-评测-成本**的全链路工程。

## 2) 可抽取资产（Definitions / Variables / Mechanisms）

### 2.1 定义（把“学习项目”当作系统）

- **知识基础设施（Knowledge Infra）**：可持续的索引、路径、题库、复现清单与工程落地笔记，让新人能在更短时间进入“可交付状态”。
- **技能栈收敛**：一个领域成熟时，学习内容会从“百科式”走向“能交付的最小全栈”（控制/数据/训练/部署/评测/优化）。

### 2.2 关键变量（后续验证会用到）

- **维护强度**：更新频率、目录增长结构是否持续可读（不是堆稿）
- **分发信号**：Star 增长、外部引用、issue/PR 贡献、被课程/团队采用
- **可复现性**：是否有 checklist、入口指引、可运行脚手架（而非纯概念）
- **职业/组织转化**：是否形成“人才管道”（被用作面试/训练/ onboarding）

### 2.3 机制（为什么它可能有价值 / 为什么可能无效）

- **有效路径**：
  - 行业进入扩张期 → 人才缺口大 → “能交付的学习路径”稀缺 → 知识库成为分发入口与能力标准
  - 评测/部署/成本成为硬约束 → 纯论文型学习不足 → 全栈材料更有复利
- **失效路径**：
  - 维护者停止更新 → 过期率上升 → 读者信任下降
  - 内容变成“堆叠百科”而非可执行路线 → 对新人帮助下降

## 3) 关键主张（Claims）与可验证预测（Predictions）

1) **VLA/具身正在从“论文理解”迁移到“工程交付”**  
   - 预测：更多资料会围绕部署、评测、硬件选型、数据管道、效率优化展开（而不只架构）。

   - 预测：出现更多“路线图 + 索引 + 题库 + 复现清单”的知识基础设施项目，并被团队复用。
