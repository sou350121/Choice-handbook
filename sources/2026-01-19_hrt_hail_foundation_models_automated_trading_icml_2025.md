## 可抽取资产（Asset Extraction）

### 定义（概念边界）

- **LOB（Limit Order Book）**：用买卖盘（Bid/Ask）与其数量刻画市场供需意图的结构；“价格”不是单一数值，而是由 LOB 定义。
- **Spread**：Best Ask 与 Best Bid 差额，显性交易成本/风险补偿。
- **Mid-price**：Best Bid 与 Best Ask 的平均值，常作为“公允价值代理”。
- **做市（Market Making / Passive Orders）**：提供流动性赚取 spread，本质是跨时间的供需匹配。
- **逆向选择（Adverse Selection）**：对手方信息更优导致做市商在不利价格变动前成交。
- **吃单（Removing Liquidity / Aggressive Orders）**：基于预测主动成交，需要更“聪明”或更“快”。
- **Smart vs Fast Pareto Frontier**：模型越复杂越聪明但推理延迟更高；系统越快越依赖低延迟与简单计算。
- **Time-based vs Event-based Tokenization**：金融市场事件非均匀（burst）；按时间切片会平滑细节，按事件切片更贴近本质但更难建模与推理。
- **有效样本量（Effective Sample Size）**：预测视界越长，样本“有效独立性”下降，信噪比更低、过拟合风险更高。

### 变量（关键量）

- **微观结构变量**：流动性、spread、mid-price、订单到达强度（burst 程度）
- **策略变量**：做市 vs 吃单、逆向选择风险、滑点
- **系统变量**：推理延迟（latency）、吞吐、算力预算（compute）、执行速度
- **建模变量**：序列长度、记忆机制（hidden state 容量与损失）、归纳偏置强度
- **目标变量**：预测 horizon（tick、分钟、小时）、信噪比、有效样本量
- **训练变量**：数据规模、参数规模、scaling law 曲线形态

### 机制（因果链 / 互动算子）

- **更高算力 → 更强模型 → 更高推理延迟** → 在低延迟场景触发“不可上线”的现实尽头
- **事件非均匀 + 时间切片** → 微观结构被抹平 → alpha 信号被稀释
- **RNN/SSM 低延迟**（$\\mathcal{O}(1)$ 推理）**+ 强归纳偏置** → 在低信噪比金融数据里，压缩历史到 hidden state 可能丢失关键依赖
- **长 horizon** → 允许大模型，但有效样本量下降 + 非平稳加剧 → 过拟合与失真风险上升
- **无完美仿真器 + 非零和 + 非平稳** → 回测无法复刻市场冲击与对手反应 → 端到端 RL 受限

### 结论（更像工程主张）

- 金融基础模型是“算法 + 数据 + 基础设施 + 延迟工程”的系统工程，不是单点模型问题。
- 需要在架构上突破推理成本（Transformer 长上下文优势 vs 延迟成本）或增强记忆能力（RNN/SSM 的信息压缩损失）。
- 在足够数据规模与算力下，scaling laws 在金融场景可能仍成立（原梳理中提到 HRT 具备 >100TB 清洗数据与大规模训练）。

### 反例/失败模式（尽头线索）

- 模型很聪明但推理太慢：alpha 存在但无法实时执行，收益被延迟与滑点吃掉
- 视界过长：信噪比与有效样本量太低导致过拟合，回测好看但实盘失效
- 仅靠回测：市场冲击与对手反应缺失导致策略崩塌（“无完美仿真器”的尽头）
- 特征工程缺失：原始流数据未被转成可学习表征，模型学习不到稳定信号

### 可验证预测（假设/推演）

- 在固定延迟预算下，事件驱动表示（event-based）会在微观结构任务上优于粗粒度时间切片（time-based）。
- 在固定 horizon 下，模型规模与数据规模提升带来的收益会呈现可预测的边际递减（scaling law 形态），但收益曲线强依赖数据清洗与非平稳处理。
