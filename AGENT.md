# AGENT.md

本文件面向在 `choice-handbook/` 中工作的自动化/AI agent 或协作者，说明写作与维护规范。

## 目标
- 把“选择”写成**可执行、可复盘、可持续迭代**的手册，而不是观点集合。
- 保持结构化与可检索：读者能快速定位“我该怎么做”。
- 允许更有趣的表达：用“未来研究员/未来回看者”的视角写，但必须**落回可证伪的信号与行动**（避免变成纯文学）。

## 内容与格式规范
- **语言**：简体中文为主；必要时保留英文关键词（如 option value / premortem）。
- **命名**：文件名使用 `snake_case.md`，不要空格/大写。
- **输入材料类型**：用户可能给采访/论文/文章/随笔/帖子/纪要/报告等；你需要能先做高质量分类，再做分析与产出。
- **叙事与历史代入（默认能力）**：
  - **未来人视角（hybrid）**：允许更强的镜头感与故事感，但每个关键推演必须落回：事实/假设/推演标签 + 可回归信号 + 可执行下一步。
  - **年代代入**：回看历史时，要写清“当时的人处在什么年代背景、能看到什么/看不到什么、有哪些可选路径与代价”；并显式区分“当时信息”与“事后上帝视角”。
- **写作模板（每篇统一）**：
  1. 概述（这篇解决什么选择/误区）
  2. 框架/模型（1–3 个，给出使用条件）
  3. **材料分类与索引（当输入来源复杂时先做）**
     - 类型：采访/论文/综述/观点文/报告/教程/讨论串/纪要等
     - 主题：属于哪个领域（可多标签）
     - 语气与可信度：第一手/二手；数据/论证/引用是否可追溯；潜在利益相关
     - 可抽取资产：定义、变量、机制、结论、反例、可验证预测、方法步骤
     - 输出去向：应进入 `principles/` 还是 `frameworks/` 还是 `cases/` 或 `worksheets/`
  4. **降维解剖（当用户输入一个学科/领域时默认执行）**
     - 第一性原理扫描：剔除类比、修辞和次级知识，只保留物理层或逻辑层的真理
     - 提取根本问题 $\\mathcal{O}$：识别核心矛盾，定义目标函数
     - Loss Function Lens：把目标写成“损失 + 约束 + proxy + 权重”（通用+ML 版见 `frameworks/05_reasoning_toolkit.md` 的 `B1`）
     - 构建根本骨架：核心变量（原子单元）与核心定律（互动算子）
     - 形式化映射：转成数学表达或逻辑伪代码（可读、可执行、可检验）
  5. **比较思考（发展史 / 对比技术 / 当下 / 未来）**（建议加入，尤其是技术与趋势话题）
     - 发展史：这类事物过去经历过哪些阶段？阶段之间的跃迁由什么驱动？
     - 对比对象：找 1–2 个“结构相似”的旧技术/旧范式（而不是表面相似）
     - 映射：把旧案例的关键变量映射到当下（供给、分发、成本、监管、生态、用户心智）
     - 未来：给 2–3 条情景（保守/基准/激进）+ 观察信号（什么发生代表哪条情景更可能）
     - 反类比：明确“哪里不像”，避免把类比当结论
  6. **尽头坐标**
     - 现实停止处：资源/约束/代价（钱/时间/精力/关系/能力/制度）
     - 两问：  
       - “本来可以发生却为什么没发生？”  
       - “已堪堪发生却退回去复归不会发生？”
     - 输出：**退出条件/止损线/停机点** + **重启条件**
  7. 短/中/长期利益（1–3 个月 / 1–3 年 / 5–10 年）
  8. MUST/SHOULD/MAY 清单
  9. **可投資標的（Long / Hedge / Short）**（当文档属于 `world_understanding/`，或明显涉及产业/趋势/资产定价时默认加入）
  10. **Plan Prompt（可直接复制给 Agent 执行）**
     - 目标：这次“小成本验证/访问/试运行”要拿到什么证据
     - 输入：文章链接/对话记录/背景约束/现有资料
     - 输出：产物格式（访谈纪要/验证结论/下一步建议/退出条件更新）
     - 步骤：按步执行（时间盒、访问名单、问题清单、判定标准）
     - 约束：隐私/不编造/不越界（只在 `choice-handbook/` 内落盘）
  11. 参考/延伸（需要时）

### world_understanding/ 的“年代镜头三段”（默认应出现）

为让内容更有趣且更“确定”，`world_understanding/` 建议显式加入并写实写满：

- `## 年代镜头（当时的人看到什么）`：当时的约束/主流信念/可见信号/盲区
- `## 分叉点（当时可选的路径与代价）`：至少 2–3 条路径，各自代价与当时会如何下注
- `## 收敛（我更确定的未来来自哪里）`：用“机制不变量 + 约束 + 观察信号”写成 3–5 条

## “可投資標的”写作标准（避免过少/不精准）

> 目标：把“世界理解”翻译成可执行的交易表达（仍然不构成投资建议）。

- **最小覆盖（默认）**：
  - **Long / Hedge / Short** 三段都要写（除非明确说明“不适用”）
  - **结构不强制**：可以分桶、可以按因子、可以按情景、也可以按“机制链条/约束”来组织；但必须让读者看得出“押什么 → 用什么工具表达”
  - **代表性集合要足够**：优先给 **ETF（篮子）+ 单票（龙头）** 的组合；避免只给 1–2 个示意票
- **精确性规则**：
  - **先说“押什么机制”**，再给标的；每个桶必须能回答“为什么是它/它暴露的是什么因子”
  - 必须写 **篮子 vs 单票** 的使用条件（不想赌单票→ETF；对某机制强信念→少量单票）
  - 必须写 **观察信号 → 标的敏感性**：信号变化时，哪些桶/哪些标的更可能受益或受损
- **可交易性与风险（简要）**：
  - 标的尽量选流动性好、能被多数账户交易的；不确定就篮子化
  - 对冲工具优先用 **指数/行业 ETF 的期权**（put/put spread），并强调限损
- **核验要求（避免写错）**：
  - 当你对 **ticker/交易所/ETF 名称/品类** 不确定时，必须先做 **web search** 核验后再写入（不要凭记忆编造）
  - 如果找不到可靠信息，宁可篮子化或删掉该标的，也不要硬写

## 索引维护（必须同步）
- 新增或改名文档：必须更新对应目录的 `README.md` 索引。
- 根入口变更：同步更新 `choice-handbook/README.md` 的“核心入口/阅读路线”。

## 质量控制（防止空泛）
- 每篇必须提供：**下一步实验**（小成本验证/访问/试运行）至少 1 个。
- 每篇必须提供：**Plan Prompt**（让“下一步实验”可直接执行，而不是停留在建议）。
- 每篇必须提供：**退出条件**（什么时候该停）至少 1 个。
- 不要编造“研究结论/数据/书中原句”。引用外部内容时要给来源（可选）。

## Plan Prompt 模板（直接复制用）

### A) 小成本验证（Experiment Prompt）
```
你是执行代理。请在不编造信息的前提下，帮我完成一次“小成本验证”。

## 背景
<我正在做的选择/文章核心观点/我想验证的假设>

## 目标（要拿到的证据）
- <证据1：是/否/阈值>
- <证据2：是/否/阈值>

## 约束
- 时间盒：<例如 60 分钟 / 7 天>
- 投入上限：<钱/时间/精力>
- 隐私：不要写入任何真实个人隐私或敏感信息

## 你要做的事
1) 把“最危险假设”写成可检验陈述（可证伪）
2) 设计最小实验（动作 + 成本 + 预期信号 + 判定标准）
3) 给出执行步骤清单（按时间顺序，细到我今天该做什么）
4) 写出退出条件与重启条件（触发标准必须可观测）
5) 输出一段可直接粘贴到 `worksheets/decision_one_pager.md` 的内容
```

### B) 访问/访谈（Interview Prompt）
```
你是访谈代理。请帮我用最小成本做一次“访问/访谈”，用来验证一个选择/观点。

## 访谈对象画像（至少 2 类）
- <例如：行业内从业者/目标岗位同岗/潜在用户/合作方>

## 目标
- 我要验证的关键假设：<...>
- 我想拿到的证据：<...>

## 你要输出
1) 访谈名单策略（怎么找人、优先顺序、每类 2–3 人）
2) 约访消息模板（短/中两版）
3) 访谈提纲（10–12 个问题，按层级：事实→机制→反例→建议）
4) 记录模板（可直接复制填写）
5) 判定标准：什么回答/信号意味着假设更可能为真/为假
6) 访谈后如何更新：下一步实验 + 退出条件/重启条件
```

### C) 试运行/影子执行（Trial Prompt）
```
你是行动规划代理。请帮我做一次“试运行”，把一个选择降到可执行的 7 天计划。

## 选择
<我要试运行的选项/路径>

## 时间盒
7 天（可调整）

## 你要输出
1) 每日计划（每天 3 个动作：最小/标准/加注版）
2) 指标与记录（每天 3 个指标：进展/成本/状态）
3) 风险与预案（最可能失败路径 + 预防动作）
4) 退出条件（停机点）+ 重启条件（证据门槛）
5) 第 7 天复盘脚本（对应 `worksheets/postmortem.md`）
```
