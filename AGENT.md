# AGENT.md

本文件面向在 `choice-handbook/` 中工作的自动化/AI agent 或协作者，说明写作与维护规范。

## 目标
- 把“选择”写成**可执行、可复盘、可持续迭代**的手册，而不是观点集合。
- 保持结构化与可检索：读者能快速定位“我该怎么做”。

## 内容与格式规范
- **语言**：简体中文为主；必要时保留英文关键词（如 option value / premortem）。
- **命名**：文件名使用 `snake_case.md`，不要空格/大写。
- **输入材料类型**：用户可能给采访/论文/文章/随笔/帖子/纪要/报告等；你需要能先做高质量分类，再做分析与产出。
- **写作模板（每篇统一）**：
  1. 概述（这篇解决什么选择/误区）
  2. 框架/模型（1–3 个，给出使用条件）
  3. **材料分类与索引（当输入来源复杂时先做）**
     - 类型：采访/论文/综述/观点文/报告/教程/讨论串/纪要等
     - 主题：属于哪个领域（可多标签）
     - 语气与可信度：第一手/二手；数据/论证/引用是否可追溯；潜在利益相关
     - 可抽取资产：定义、变量、机制、结论、反例、可验证预测、方法步骤
     - 输出去向：应进入 `principles/` 还是 `frameworks/` 还是 `cases/` 或 `worksheets/`
  4. **降维解剖（当用户输入一个学科/领域时默认执行）**
     - 第一性原理扫描：剔除类比、修辞和次级知识，只保留物理层或逻辑层的真理
     - 提取根本问题 \(\\mathcal{O}\\)：识别核心矛盾，定义目标函数
     - 构建根本骨架：核心变量（原子单元）与核心定律（互动算子）
     - 形式化映射：转成数学表达或逻辑伪代码（可读、可执行、可检验）
  5. **比较思考（发展史 / 对比技术 / 当下 / 未来）**（建议加入，尤其是技术与趋势话题）
     - 发展史：这类事物过去经历过哪些阶段？阶段之间的跃迁由什么驱动？
     - 对比对象：找 1–2 个“结构相似”的旧技术/旧范式（而不是表面相似）
     - 映射：把旧案例的关键变量映射到当下（供给、分发、成本、监管、生态、用户心智）
     - 未来：给 2–3 条情景（保守/基准/激进）+ 观察信号（什么发生代表哪条情景更可能）
     - 反类比：明确“哪里不像”，避免把类比当结论
  5. **尽头坐标（唐诺）**
     - 现实停止处：资源/约束/代价（钱/时间/精力/关系/能力/制度）
     - 两问：  
       - “本来可以发生却为什么没发生？”  
       - “已堪堪发生却退回去复归不会发生？”
     - 输出：**退出条件/止损线/停机点** + **重启条件**
  6. 短/中/长期利益（1–3 个月 / 1–3 年 / 5–10 年）
  7. MUST/SHOULD/MAY 清单
  8. 练习（可复制填写）
  9. 参考/延伸（需要时）

## 索引维护（必须同步）
- 新增或改名文档：必须更新对应目录的 `README.md` 索引。
- 根入口变更：同步更新 `choice-handbook/README.md` 的“核心入口/阅读路线”。

## 质量控制（防止空泛）
- 每篇必须提供：**下一步实验**（小成本验证/访问/试运行）至少 1 个。
- 每篇必须提供：**退出条件**（什么时候该停）至少 1 个。
- 不要编造“研究结论/数据/书中原句”。引用外部内容时要给来源（可选）。
